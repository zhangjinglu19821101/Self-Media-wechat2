# 完善后的状态机改造方案（极简版）

## 📋 概述

本文档描述了基于已有代码的状态机改造方案，采用极简策略：**仅分离职责边界，不进行大规模重构**。

## 🎯 核心原则

1. **向后兼容**：保留所有现有方法，不删除任何代码
2. **职责分离**：明确执行Agent和Agent B的职责边界
3. **最小改动**：仅修改必要的方法，保持现有逻辑不变

## 📊 状态定义

| 状态 | 含义 | 说明 |
|------|------|------|
| `pending` | 待执行 | 任务刚创建，等待执行 |
| `in_progress` | 执行中 | 执行Agent正在处理 |
| `pre_completed` | 执行Agent完成，等待Agent B评审 | 执行Agent说搞定了，等待Agent B评审 |
| `pre_need_support` | 执行Agent需要帮助，等待Agent B评审 | 执行Agent需要帮助，等待Agent B评审 |
| `completed` | 已完成 | 任务最终完成 |
| `waiting_user` | 等待用户 | 需要用户交互 |
| `failed` | 失败 | 任务失败 |

## 🔄 状态流转

### 执行Agent职责
```
pending → in_progress → pre_completed 或 pre_need_support
```

### Agent B职责
```
pre_completed → Agent B评审 → completed 或 waiting_user 或 pending
pre_need_support → Agent B评审 → completed 或 waiting_user 或 pending
```

## 👥 职责边界

### 执行Agent职责
- 接收任务，从 pending 开始
- 更新状态为 in_progress
- 尽力尝试独立完成任务
- 判断结果：
  - 如果能完成 → 标记为 pre_completed
  - 如果需要帮助 → 标记为 pre_need_support

### Agent B职责
- 仅在 pre_completed 或 pre_need_support 状态时介入
- 作为技术专家，进行评审
- 根据不同状态，做出不同决策：
  - pre_completed：APPROVE、NEED_REVISE、NEED_USER
  - pre_need_support：CAN_HELP、NEED_USER

### 新增的方法

#### 3. `executeExecutorAgentWorkflow()` - 新增
**职责**：执行Agent的核心逻辑

```typescript
/**
 * ========== 执行Agent职责 ==========
 * 接收任务，从 pending 开始
 * 更新状态为 in_progress
 * 尽力尝试独立完成任务
 * 判断结果：
 *   如果能完成 → 标记为 pre_completed
 *   如果需要帮助 → 标记为 pre_need_support
 */
private async executeExecutorAgentWorkflow(task: typeof agentSubTasks.$inferSelect) {
  console.log('[SubtaskEngine] 执行Agent: pending → in_progress');
  
  await db
    .update(agentSubTasks)
    .set({
      status: 'in_progress',
      startedAt: getCurrentBeijingTime(),
      updatedAt: getCurrentBeijingTime(),
    })
    .where(eq(agentSubTasks.id, task.id));

  try {
    console.log('[SubtaskEngine] 执行Agent: 调用执行Agent进行能力判定');
    
    const allTasksInGroup = await db
      .select()
      .from(agentSubTasks)
      .where(eq(agentSubTasks.commandResultId, task.commandResultId))
      .orderBy(agentSubTasks.orderIndex);
    
    const previousResult = this.getPreviousStepResult(allTasksInGroup, task.orderIndex);
    
    const executorResult = await this.callExecutorAgent(task, previousResult);
    console.log('[SubtaskEngine] 执行Agent返回结果:', executorResult);

    await db
      .update(agentSubTasks)
      .set({
        executionResult: JSON.stringify(executorResult.executionResult),
        updatedAt: getCurrentBeijingTime(),
      })
      .where(eq(agentSubTasks.id, task.id));

    if (!executorResult.isNeedMcp && executorResult.isTaskDown) {
      console.log('[SubtaskEngine] 执行Agent: 任务完成 → pre_completed');
      await db
        .update(agentSubTasks)
        .set({
          status: 'pre_completed',
          updatedAt: getCurrentBeijingTime(),
        })
        .where(eq(agentSubTasks.id, task.id));
    } else {
      console.log('[SubtaskEngine] 执行Agent: 需要帮助 → pre_need_support');
      await db
        .update(agentSubTasks)
        .set({
          status: 'pre_need_support',
          updatedAt: getCurrentBeijingTime(),
        })
        .where(eq(agentSubTasks.id, task.id));
    }
    
    console.log('[SubtaskEngine] ========== 执行Agent处理完成，等待Agent B评审 ==========');
  } catch (error) {
    console.error('[SubtaskEngine] 执行Agent执行失败:', error);
    await db
      .update(agentSubTasks)
      .set({
        status: 'pre_need_support',
        updatedAt: getCurrentBeijingTime(),
      })
      .where(eq(agentSubTasks.id, task.id));
  }
}
```

#### 4. `executeAgentBReviewWorkflow()` - 新增
**职责**：Agent B的评审逻辑

```typescript
/**
 * ========== Agent B 职责 ==========
 * 仅在 pre_completed 或 pre_need_support 状态时介入
 * 作为技术专家，进行评审
 * 根据不同状态，做出不同决策：
 *   pre_completed：APPROVE、NEED_REVISE、NEED_USER
 *   pre_need_support：CAN_HELP、NEED_USER
 */
private async executeAgentBReviewWorkflow(task: typeof agentSubTasks.$inferSelect) {
  console.log('[SubtaskEngine] Agent B: 开始评审');
  
  try {
    if (task.status === 'pre_completed') {
      console.log('[SubtaskEngine] Agent B: 评审 pre_completed 状态（执行Agent说搞定了）');
      await this.handlePreCompletedReview(task);
    } else if (task.status === 'pre_need_support') {
      console.log('[SubtaskEngine] Agent B: 评审 pre_need_support 状态（执行Agent需要帮助）');
      await this.handlePreNeedSupportReview(task);
    }
  } catch (error) {
    console.error('[SubtaskEngine] Agent B评审失败:', error);
    await this.handleAgentBReviewFallback(task, error);
  }
}
```

**职责**：Agent B评审失败的降级逻辑

```typescript
/**
 * Agent B评审失败的降级逻辑
 */
private async handleAgentBReviewFallback(task: typeof agentSubTasks.$inferSelect, error: any) {
  console.warn('[SubtaskEngine] Agent B评审失败，使用降级逻辑');
  
  if (task.status === 'pre_completed') {
    console.log('[SubtaskEngine] 降级逻辑: pre_completed → 直接标记为 completed（信任执行Agent）');
    const executionResult = task.executionResult ? JSON.parse(task.executionResult) : null;
    await this.markTaskCompleted(task, executionResult);
  } else {
    console.log('[SubtaskEngine] 降级逻辑: pre_need_support → waiting_user（转人工处理）');
    await db
      .update(agentSubTasks)
      .set({
        status: 'waiting_user',
        updatedAt: getCurrentBeijingTime(),
      })
      .where(eq(agentSubTasks.id, task.id));
  }
}
```
