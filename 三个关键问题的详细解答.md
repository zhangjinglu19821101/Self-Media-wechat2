## 问题 1：执行 agent 执行后返回了什么？

### 1.1 执行 agent 的返回类型

执行 agent 返回的是 `ExecutorAgentResult` 类型：

```typescript
interface ExecutorAgentResult {
  isNeedMcp: boolean;        // 是否需要 MCP 介入
  problem?: string;           // 遇到的具体问题
  capabilityType?: string;    // 需要的能力类型
  executionResult?: any;      // 执行结果（如果不需要 MCP）
  isTaskDown: boolean;        // 任务是否已完成
}
```


## 问题 3：Agent B 能通过 capability + 执行 agent 的具体指令 + 执行 agent 的执行结果，判断出来能不能调用本地合规校验的 MCP 服务吗？

### 3.1 答案是：**能！而且代码中已经实现了这个逻辑！**

### 3.2 Agent B 判断的三个关键信息

Agent B 确实会综合以下三个信息来判断：

1. **capability（可用的 MCP 能力清单）**
2. **执行 agent 的具体指令（任务信息）**
3. **执行 agent 的执行结果（executorResult）**

### 3.3 具体实现代码

让我展示代码中是如何实现的：

#### 3.3.1 第一步：判断是否需要两阶段流程

```typescript
/**
 * 判断是否需要"先合规检查，后上传"的两阶段流程
 */
private needsTwoPhaseProcess(
  task: typeof agentSubTasks.$inferSelect, 
  executorResult: ExecutorAgentResult
): boolean {
  // 条件 1：来自 insurance-d
  const isInsuranceD = task.fromParentsExecutor === 'insurance-d';
  
  // 从 metadata 中获取 taskType
  const taskType = (task.metadata as any)?.taskType || '';
  
  // 条件 2：任务类型涉及内容发布
  const isContentPublishingTask = 
    (taskType &amp;&amp; (taskType.includes('article') || taskType.includes('content'))) ||
    (task.taskTitle &amp;&amp; (task.taskTitle.includes('发布') || task.taskTitle.includes('公众号') || task.taskTitle.includes('上传'))) ||
    executorResult.capabilityType === 'platform_publish';
  
  // 条件 3：不是单纯的合规审核任务
  const isComplianceOnlyTask = 
    (task.taskTitle &amp;&amp; (task.taskTitle.includes('合规') || task.taskTitle.includes('审核') || task.taskTitle.includes('校验'))) ||
    executorResult.capabilityType === 'content_audit';
  
  return isInsuranceD &amp;&amp; isContentPublishingTask &amp;&amp; !isComplianceOnlyTask;
}
```

**这里用到的信息**：
- ✅ 执行 agent 的具体指令：`task.taskTitle`、`task.metadata.taskType`
- ✅ 执行 agent 的执行结果：`executorResult.capabilityType`
