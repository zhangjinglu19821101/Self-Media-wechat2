### 测试场景与 agent_sub_tasks_step_history 数据映射

| 场景 | decision.type | execution_result.status | mcp_attempts | 特殊字段 |
|------|---------------|------------------------|--------------|----------|
| 首次成功 | `COMPLETE` | `success` | 1条成功记录 | - |
| 重试后成功 | `COMPLETE` | `success` | N条(失败+成功) | execution_summary |
| 重试后失败 | `FAILED` | `failed` | N条失败记录 | failed_details |
| 首次需用户确认 | `NEED_USER` | `waiting_user` | 空数组 | pending_key_fields |
| 执行后需用户确认 | `NEED_USER` | `waiting_user` | N条记录 | pending_key_fields |
| 最大迭代次数 | `FAILED` | `failed` | N条记录 | ext_info.failed_details |
| 用户确认后完成 | `COMPLETE` | `success` | 1条成功记录 | - |

## 测试方法

### 0. 数据清理（测试前准备）

#### API 端点
```
POST /api/test/cleanup-data
```

#### 功能说明
- 清理指定的测试相关表数据
- 避免历史数据干扰新的测试结果
- **建议在运行系统测试前执行**

#### 请求示例
```bash
curl -s -X POST 'http://localhost:5000/api/test/cleanup-data' \
  -H "Content-Type: application/json" \
  -d '{"tables":["agent_sub_tasks","daily_task"]}' | jq .
```

#### 支持清理的表
- `agent_sub_tasks` - 子任务表
- `daily_task` - 日工作任务表
- `agent_reports` - Agent 报告表
- `agent_interactions` - Agent 交互记录表

#### 响应示例
```json
{
  "success": true,
  "message": "✅ 数据清理完成",
  "cleanedTables": ["agent_sub_tasks", "daily_task"],
  "failedTables": [],
  "details": {
    "agent_sub_tasks": { "success": true },
    "daily_task": { "success": true }
  }
}
```

## 执行流程

### 推荐流程（一键执行）

#### Step 1: 清理历史数据（可选但推荐）
```bash
curl -s -X POST 'http://localhost:5000/api/test/cleanup-data' \
  -H "Content-Type: application/json" \
  -d '{"tables":["agent_sub_tasks","daily_task"]}' | jq .
```

#### Step 2: 执行所有测试用例
```bash
curl -s 'http://localhost:5000/api/test/run-all-tests' | jq .
```

#### Step 3: 查看最终结果（如需要）
使用 Step 2 返回的 `testGroupId` 查询：
```bash
curl -s 'http://localhost:5000/api/test/run-all-tests?query=true&testGroupId={testGroupId}' | jq .
```

### 手动流程（分步执行）
1. **创建测试数据**：`POST /api/test/e2e-functional-tests`
2. **查询任务ID**：`GET /api/test/e2e-functional-tests`
3. **执行子任务**：`POST /api/test/e2e-functional-tests` (body: `{action: "execute", taskId}`)
4. **重复执行**：直到任务进入终态
5. **查看结果**：`GET /api/test/e2e-functional-tests?query=true&taskId={taskId}`

### API 列表
| API | 方法 | 用途 | 模式 |
|-----|------|------|------|
| `/api/test/cleanup-data` | POST | 清理测试数据 | 一键执行 |
| `/api/test/run-all-tests` | GET | 一键执行所有测试 | 一键执行 |
| `/api/test/e2e-functional-tests` | POST/GET | 创建/查询/执行子任务 | 手动模式 |

## 注意事项

1. **数据清理**: 建议运行系统测试前清理相关表数据，避免历史数据干扰测试结果
   ```bash
   # 清理 agent_sub_tasks 和 daily_task 表
   curl -s -X POST 'http://localhost:5000/api/test/cleanup-data' -H "Content-Type: application/json" -d '{"tables":["agent_sub_tasks","daily_task"]}'
   ```

2. **执行顺序**: 测试任务按 `orderIndex` 顺序执行，前序任务未完成时后续任务不会启动

3. **超时处理**: 任务执行超过10分钟会触发超时检测流程

4. **重试机制**: MCP 调用失败会自动重试，最多3次

5. **最大迭代**: Agent B 决策循环最多5次

6. **定时任务**: 如果手动执行后任务仍在 `in_progress`，等待定时任务调度（每2分钟）
