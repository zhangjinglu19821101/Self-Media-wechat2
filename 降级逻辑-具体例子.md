# 降级逻辑 - 具体场景例子

## 📋 核心概念回顾

**降级逻辑 = 安全网**

```
try {
  // 新逻辑：Agent B 评审（可能会出错）
  const reviewResult = await this.agentBReviewer.reviewPreCompleted(task);
  await this.processPreCompletedReviewResult(task, reviewResult);
  
} catch (error) {
  // ❌ 新逻辑出错了！
  
  // ✅ 自动降级：使用原来的逻辑
  await this.handlePreCompletedFallback(task);
}
```

## 🎯 场景 1：Agent B API 报错（网络问题）

### 场景描述
- 任务状态：`pre_completed`
- 执行Agent（insurance-d）已经完成了任务
- 正准备调用 Agent B 进行评审
- 但是网络出问题了，Agent B API 调用失败

### 代码执行流程

```typescript
// 1. 状态机开始处理 pre_completed
private async handlePreCompletedStatus(task: typeof agentSubTasks.$inferSelect) {
  console.log(`[状态机] 处理 pre_completed: ${task.id}`);
  
  try {
    // 2. 尝试调用 Agent B 进行评审
    console.log(`[状态机] 调用 Agent B 进行评审...`);
    
    // ❌ 这里出错了！网络问题，Agent B API 调用失败
    const reviewResult = await this.agentBReviewer.reviewPreCompleted(task);
    // ↑↑↑ 这里会抛出异常 ↑↑↑
    
    // 下面这行永远不会执行
    await this.processPreCompletedReviewResult(task, reviewResult);
    
  } catch (error) {
    // ✅ 捕获到异常！
    console.error(`[状态机] Agent B 评审失败，使用降级逻辑:`, error);
    // 输出类似：[状态机] Agent B 评审失败，使用降级逻辑: NetworkError: timeout
    
    // ✅ 自动降级：使用原来的逻辑
    await this.handlePreCompletedFallback(task);
  }
}

// 3. 降级逻辑：原来的逻辑（直接标记为 completed）
private async handlePreCompletedFallback(task: typeof agentSubTasks.$inferSelect) {
  console.log(`[状态机] 使用降级逻辑：直接标记为 completed`);
  
  // 就是原来的逻辑，不经过 Agent B 评审
  await this.markTaskCompleted(
    task,
    task.executionResult ? JSON.parse(task.executionResult) : { success: true }
  );
}
```

### 实际效果

| 项目 | 说明 |
|------|------|
| **如果没有降级逻辑** | 任务卡在 pre_completed 状态，永远无法完成 ❌ |
| **有降级逻辑** | 任务直接标记为 completed，流程继续 ✅ |

### 日志输出

```

[状态机] 调用 Agent B 进行评审...
[Agent B 评审器] 开始评审 pre_completed: task-123
❌ [Agent B 评审器] 评审失败: NetworkError: timeout
❌ [状态机] Agent B 评审失败，使用降级逻辑: NetworkError: timeout
✅ [状态机] 使用降级逻辑：直接标记为 completed
✅ [状态机] 任务已标记为 completed: task-123
```

## 🎯 场景 2：评审响应解析失败（JSON 格式错误）

### 场景描述
- 任务状态：`pre_completed`
- Agent B API 调用成功了
- 但是返回的响应格式不对，不是 valid JSON
- 解析响应时出错了

### 代码执行流程

```typescript
class AgentBReviewer {
  async reviewPreCompleted(task: typeof agentSubTasks.$inferSelect): Promise<AgentBReviewResult> {
    try {
      // 1. 调用 Agent B（成功）
      const response = await callLLM('B', ...);
      // 假设返回的是：
      // "我觉得这个结果不错，可以批准"（纯文本，不是 JSON）
      
      // 2. 尝试解析响应
      const reviewResult = this.parseReviewResponse(response);
      
      return reviewResult;
      
    } catch (error) {
      // ❌ 解析失败！
      console.error('[Agent B 评审器] 解析响应失败:', error);
      
      // 返回默认决策
      return {
        decision: 'APPROVE',
        reasoning: '解析评审响应失败，使用默认决策',
      };
    }
  }
  
  private parseReviewResponse(response: string): AgentBReviewResult {
    try {
      const jsonMatch = response.match(/\{[\s\S]*\}/);
      if (jsonMatch) {
        // ❌ 这里会失败，因为响应是纯文本，没有 JSON
        const parsed = JSON.parse(jsonMatch[0]);
        return {
          decision: parsed.decision,
          reasoning: parsed.reasoning,
          data: parsed.data,
        };
      }
    } catch (e) {
      // ❌ 抛出异常
      console.error('[Agent B 评审器] 解析响应失败:', e);
      throw e; // 重新抛出，让外层处理
    }
    
    throw new Error('无法解析响应');
  }
}

// 外层捕获
private async handlePreCompletedStatus(task: typeof agentSubTasks.$inferSelect) {
  try {
    const reviewResult = await this.agentBReviewer.reviewPreCompleted(task);
    // ↑↑↑ 这里会收到默认决策 APPROVE ↑↑↑
    
    await this.processPreCompletedReviewResult(task, reviewResult);
    
  } catch (error) {
    // 这里不会触发，因为 AgentBReviewer 内部已经处理了
    // 但如果再出错，还是会降级
    await this.handlePreCompletedFallback(task);
  }
}
```

### 实际效果

| 项目 | 说明 |
|------|------|
| **Agent B 响应** | "我觉得这个结果不错，可以批准"（纯文本） |
| **解析结果** | 失败，不是 valid JSON |
| **降级处理** | AgentBReviewer 返回默认决策 APPROVE |
| **最终结果** | 任务标记为 completed ✅ |

### 日志输出

```
[状态机] 处理 pre_completed: task-123
[状态机] 调用 Agent B 进行评审...
[Agent B 评审器] 开始评审 pre_completed: task-123
[Agent B 评审器] 调用 LLM 成功
❌ [Agent B 评审器] 解析响应失败: SyntaxError: Unexpected token '我'
⚠️  [Agent B 评审器] 使用默认决策: APPROVE
[状态机] 处理评审结果: APPROVE
✅ [状态机] Agent B 批准，标记为 completed
```

## 🎯 场景 3：数据库操作失败（更新状态时出错）

### 场景描述
- 任务状态：`pre_completed`
- Agent B 评审成功了，返回 NEED_REVISE
- 准备更新任务状态为 `pending`
- 但是数据库出问题了，更新失败

### 代码执行流程

```typescript
private async processPreCompletedReviewResult(
  task: typeof agentSubTasks.$inferSelect,
  reviewResult: AgentBReviewResult
) {
  console.log(`[状态机] 处理评审结果: ${reviewResult.decision}`);
  
  switch (reviewResult.decision) {
    case 'NEED_REVISE':
      // 需要修改：让执行Agent重新执行
      console.log(`[状态机] Agent B 要求修改，回到 pending 状态`);
      
      try {
        // ❌ 这里出错了！数据库连接失败
        await db
          .update(agentSubTasks)
          .set({
            status: 'pending',
            metadata: {
              ...task.metadata,
              retryCount: (task.metadata?.retryCount || 0) + 1,
              agentBRevision: reviewResult.data?.revisionInstructions,
            },
            updatedAt: getCurrentBeijingTime(),
          })
          .where(eq(agentSubTasks.id, task.id));
        // ↑↑↑ 这里会抛出异常 ↑↑↑
        
      } catch (dbError) {
        // ❌ 数据库操作失败
        console.error(`[状态机] 数据库更新失败:`, dbError);
        throw dbError; // 重新抛出，让外层处理
      }
      break;
  }
}

private async handlePreCompletedStatus(task: typeof agentSubTasks.$inferSelect) {
  try {
    const reviewResult = await this.agentBReviewer.reviewPreCompleted(task);
    await this.processPreCompletedReviewResult(task, reviewResult);
    // ↑↑↑ 这里会抛出数据库异常 ↑↑↑
    
  } catch (error) {
    // ✅ 捕获到异常！
    console.error(`[状态机] 处理失败，使用降级逻辑:`, error);
    // 输出类似：[状态机] 处理失败，使用降级逻辑: DatabaseError: connection failed
    
    // ✅ 自动降级：使用原来的逻辑
    await this.handlePreCompletedFallback(task);
  }
}
```

### 实际效果

| 项目 | 说明 |
|------|------|
| **Agent B 决策** | NEED_REVISE（需要修改） |
| **数据库操作** | 更新状态为 pending 时失败 ❌ |
| **降级处理** | 直接标记为 completed ✅ |
| **最终结果** | 任务完成，流程继续 |

### 日志输出

```

[状态机] 调用 Agent B 进行评审...
[Agent B 评审器] 评审完成: { decision: 'NEED_REVISE', reasoning: '...' }
[状态机] 处理评审结果: NEED_REVISE
[状态机] Agent B 要求修改，回到 pending 状态
❌ [状态机] 数据库更新失败: DatabaseError: connection failed
❌ [状态机] 处理失败，使用降级逻辑: DatabaseError: connection failed
✅ [状态机] 使用降级逻辑：直接标记为 completed
✅ [状态机] 任务已标记为 completed: task-123
```

### 场景描述
- 任务状态：`pre_need_support`
- 执行Agent（insurance-d）需要帮助
- 正准备调用 Agent B 进行评审
- 但是 Agent B API 报错了

### 代码执行流程

```typescript
private async handlePreNeedSupportStatus(task: typeof agentSubTasks.$inferSelect) {
  console.log(`[状态机] 处理 pre_need_support: ${task.id}`);
  
  try {
    // 1. 调用 Agent B 进行评审
    console.log(`[状态机] 调用 Agent B 进行评审...`);

    const reviewResult = await this.agentBReviewer.reviewPreNeedSupport(task);

  }
```

## 🎯 核心思想

### 没有降级逻辑的风险

```
新逻辑出错 → 任务卡住 → 流程中断 → 用户投诉 ❌
```

### 有降级逻辑的保障

```
新逻辑出错 → 自动降级 → 使用原有逻辑 → 流程继续 ✅
```

---

## 💡 一句话总结

**降级逻辑就是：新逻辑搞不定时，用老办法兜底，确保流程不中断！** 🛡️
