# 稳健优化建议：渐进式实现 Agent B 评审逻辑

## 🎯 核心理念

**"小步快跑，风险可控"**

- ✅ 每次只改一小部分
- ✅ 保持向后兼容
- ✅ 可以随时回退
- ✅ 增加详细日志，便于监控

---

## 📋 方案一：渐进式实现（推荐）

### 阶段 3：小范围生效 - 只对特定任务生效（中高风险）

**目标**：选择一小部分任务，让 Agent B 评审生效

**改动点**：
```typescript
// 新增：配置项
const ENABLE_AGENT_B_REVIEW = false; // 可以通过环境变量控制
const AGENT_B_REVIEW_TASK_IDS = new Set<string>([]); // 特定任务ID列表

private async handlePreCompletedStatus(task: typeof agentSubTasks.$inferSelect) {
  console.log(`[状态机] 处理 pre_completed: ${task.id}`);
  
  // ========== 检查是否启用 Agent B 评审 ==========
  const shouldUseAgentBReview = ENABLE_AGENT_B_REVIEW || AGENT_B_REVIEW_TASK_IDS.has(task.id);
  
  if (shouldUseAgentBReview) {
    console.log(`[状态机] [正式模式] 启用 Agent B 评审`);
    
    const reviewResult = await this.callAgentBForReview(task, 'pre_completed');
    console.log(`[状态机] [正式模式] Agent B 评审结果:`, reviewResult);
    
    // ========== 根据评审结果改变状态 ==========
    switch (reviewResult.decision) {
      case 'APPROVE':
        await this.markTaskCompleted(task, task.executionResult ? JSON.parse(task.executionResult) : { success: true });
        console.log(`[状态机] Agent B 批准，标记为 completed`);
        break;
        
      case 'NEED_REVISE':
        // 让执行Agent重新执行
        await db
          .update(agentSubTasks)
          .set({
            status: 'pending', // 回到 pending 状态
            metadata: {
              ...task.metadata,
              agentBReview: reviewResult, // 保存 Agent B 的评审意见
            },
            updatedAt: getCurrentBeijingTime(),
          })
          .where(eq(agentSubTasks.id, task.id));
        console.log(`[状态机] Agent B 要求修改，回到 pending 状态`);
        break;
        
      case 'NEED_USER':
        await db
          .update(agentSubTasks)
          .set({
            status: 'waiting_user',
            metadata: {
              ...task.metadata,
              agentBReview: reviewResult,
            },
            updatedAt: getCurrentBeijingTime(),
          })
          .where(eq(agentSubTasks.id, task.id));
        console.log(`[状态机] Agent B 需要用户确认，标记为 waiting_user`);
        break;
    }
  } else {
    // ========== 保持原有逻辑 ==========
    console.log(`[状态机] [降级模式] 不使用 Agent B 评审，保持原有逻辑`);
    await this.markTaskCompleted(task, task.executionResult ? JSON.parse(task.executionResult) : { success: true });
    console.log(`[状态机] 直接标记为 completed`);
  }
}
```

**风险**：🟠 中高（会改变状态流转，但只对特定任务）

**好处**：
- 可以控制风险范围
- 出问题只影响特定任务
- 可以随时切换回原有逻辑

## 📋 方案三：混合模式（折中方案）

### 核心思路

**pre_completed 保持简单，pre_need_support 实现完整逻辑**

理由：
- `pre_completed` 是执行Agent说搞定了，风险较低，可以保持简单
- `pre_need_support` 是执行Agent需要帮助，更需要 Agent B 介入

```typescript
private async handlePreCompletedStatus(task: typeof agentSubTasks.$inferSelect) {
  // 保持简单：直接标记为完成
  await this.markTaskCompleted(task, task.executionResult ? JSON.parse(task.executionResult) : { success: true });
}

private async handlePreNeedSupportStatus(task: typeof agentSubTasks.$inferSelect) {
  // 实现完整的 Agent B 评审逻辑
  const reviewResult = await this.callAgentBForReview(task, 'pre_need_support');
  
  switch (reviewResult.decision) {
    case 'CAN_HELP':
      // Agent B 可以提供帮助，让执行Agent重新执行
      await db
        .update(agentSubTasks)
        .set({
          status: 'pending',
          metadata: {
            ...task.metadata,
            agentBHelp: reviewResult.data,
          },
          updatedAt: getCurrentBeijingTime(),
        })
        .where(eq(agentSubTasks.id, task.id));
      break;
      
    case 'NEED_USER':
      await db
        .update(agentSubTasks)
        .set({
          status: 'waiting_user',
          updatedAt: getCurrentBeijingTime(),
        })
        .where(eq(agentSubTasks.id, task.id));
      break;
      
    case 'CANNOT_HELP':
      await db
        .update(agentSubTasks)
        .set({
          status: 'failed',
          executionResult: JSON.stringify({ error: reviewResult.reasoning }),
          updatedAt: getCurrentBeijingTime(),
        })
        .where(eq(agentSubTasks.id, task.id));
      break;
  }
}
```

**优点**：
- ✅ 改动范围可控
- ✅ 只针对需要的场景
- ✅ 风险较低
