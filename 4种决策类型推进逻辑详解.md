## 文档信息
- **版本**: v1.0

## 一、核心流程图（总览）

```
executeCompleteWorkflow()
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│              Agent B 循环决策（最多 5 次）                  │
└─────────────────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│              switch (agentBDecision.type)                   │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐  │
│  │  COMPLETE    │  │  NEED_USER   │  │   FAILED     │  │
│  │    ↓ return  │  │    ↓ return  │  │    ↓ return  │  │
│  └──────────────┘  └──────────────┘  └──────────────┘  │
│                                                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │                  EXECUTE_MCP                         │  │
│  │                     ↓ continue                        │  │
│  │         继续下一轮决策（Agent B 再次判断）          │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 二、4 种决策类型详细推进逻辑

### 2.1 COMPLETE - 任务完成

#### 流程图

```
Agent B 决策: type = "COMPLETE"
    │
    ▼
handleCompleteDecision()
    │
    ├─ 1. 创建交互记录（response）
    │   └─ 记录: Agent B 的最终决策
    │   └─ 包含: mcp_attempts, execution_summary 等
    │
    ├─ 2. markTaskCompleted()
    │   └─ 更新数据库: status = "completed"
    │   └─ 设置: completedAt = 当前时间
    │
    └─ 3. return ← 结束整个 executeCompleteWorkflow()
              ↓
         任务推进停止
```

#### 代码逻辑

```typescript
case 'COMPLETE':
  await this.handleCompleteDecision(...);
  return;  // ← 关键：直接 return，结束整个流程！
```

#### handleCompleteDecision() 详细步骤

**文件位置**: `src/lib/services/subtask-execution-engine.ts`  
**行号**: 1360-1430

```typescript
private async handleCompleteDecision(...) {
  // 步骤 1: 创建交互记录（response，interactNum = iteration）
  await this.createInteractionStep(
    task.commandResultId,
    task.orderIndex,
    'response',              // ← 类型: response
    iteration,               // ← 交互次数
    'agent B',               // ← 发起方: agent B
    {
      interact_type: 'response',
      consultant: task.fromParentsExecutor,
      responder: 'agent B',
      question: executorResult,
      response: {
        decision: {
          type: decision.type,           // "COMPLETE"
          reason_code: decision.reasonCode,
          reasoning: decision.reasoning,
          final_conclusion: decision.context.executionSummary,
        },
        mcp_attempts: mcpExecutionHistory,      // ← 完整 MCP 历史
        user_interactions: userInteractions,      // ← 用户交互历史
        execution_summary: {
          total_mcp_attempts: mcpExecutionHistory.length,
          successful_mcp_attempts: ...,
          failed_mcp_attempts: ...,
          // ... 更多统计信息
        }
      },
      execution_result: { status: 'success' },
      ext_info: {
        step: 'agent_b_decision_complete',
        iteration,
        completion_result: decision.data?.completionResult,
      }
    }
  );

  // 步骤 2: 标记任务完成
  await this.markTaskCompleted(
    task, 
    decision.data?.completionResult || { success: true }
  );
  // markTaskCompleted 内部会:
  // - 更新 agent_subTasks 表: status = "completed"
  // - 设置 completedAt = getCurrentBeijingTime()
}
```

#### 推进影响

| 项目 | 影响 |
|-----|------|
| **任务状态** | `status` 从 `in_progress` → `completed` |
| **流程继续** | ❌ **停止** - `return` 结束 `executeCompleteWorkflow()` |
| **后续触发** | 无需重新触发，任务已完成 |
| **交互记录** | ✅ 创建 1 条 `response` 类型记录 |

### 2.2 NEED_USER - 需要用户介入（重点详解！）

#### 完整流程图

```
Agent B 决策: type = "NEED_USER"
    │
    ▼
handleNeedUserDecision()
    │
    ├─ 1. 创建交互记录 2（response）
    │   └─ type: "response"
    │   └─ interactNum: iteration
    │   └─ content: "需要用户介入"（简化，不暴露详细字段）
    │
    ├─ 2. 创建交互记录 3（request）← 关键！真正的用户问题
    │   └─ type: "request"
    │   └─ interactNum: iteration + 1  ← 次数 +1！
    │   └─ content: 包含完整的 pending_key_fields, 
    │                 available_solutions, prompt_message
    │
    ├─ 3. 更新任务状态
    │   └─ agent_subTasks.status = "waiting_user"
    │
    └─ 4. return ← 结束 executeCompleteWorkflow()
              ↓
         任务推进暂停 ⏸️
         等待用户反馈后，才会重新触发流程！
```

#### 代码逻辑

```typescript
case 'NEED_USER':
  await this.handleNeedUserDecision(...);
  return;  // ← 关键：直接 return，结束流程，等待用户！
```

#### handleNeedUserDecision() 详细步骤

**文件位置**: `src/lib/services/subtask-execution-engine.ts`  
**行号**: 1435-1520

```typescript
private async handleNeedUserDecision(...) {
  console.log('[SubtaskEngine] 处理NEED_USER决策');

  // ========== 记录 2：Agent B 回答执行 Agent（response）==========
  // 注意：这里只说"需要用户介入"，不放 pending_key_fields 等详细字段
  await this.createInteractionStep(
    task.commandResultId,
    task.orderIndex,
    'response',              // ← 类型: response
    iteration,               // ← 交互次数: iteration
    'agent B',               // ← 发起方: agent B
    {
      interact_type: 'response',
      consultant: task.fromParentsExecutor,
      responder: task.fromParentsExecutor,
      question: executorResult,
      response: {
        decision: {
          type: decision.type,           // "NEED_USER"
          reason_code: decision.reasonCode,
          reasoning: decision.reasoning,
          final_conclusion: '需要用户介入',  // ← 简化描述
        },
        mcp_attempts: mcpExecutionHistory,
        user_interactions: userInteractions,
      },
      execution_result: { status: 'need_user' },
      ext_info: {
        step: 'agent_b_decision_need_user',
        iteration,
      }
    }
  );

  // ========== 记录 3：Agent B 问用户（request）← 关键！真正的用户问题 ==========
  console.log('[SubtaskEngine] 记录交互3：Agent B 问用户（request）');
  await this.createInteractionStep(
    task.commandResultId,
    task.orderIndex,
    'request',               // ← 类型: request（问用户）
    iteration + 1,           // ← 交互次数: iteration + 1（重要！）
    'agent B',               // ← 发起方: agent B
    {
      interact_type: 'request',
      consultant: 'agent B',
      responder: 'human',    // ← 接收方: human（用户）
      question: {
        // ← 关键：完整的用户问题数据！
        pending_key_fields: decision.data?.pendingKeyFields || [],
        available_solutions: decision.data?.availableSolutions || [],
        prompt_message: decision.data?.promptMessage,
      },
      response: '',           // ← 空，等待用户填写
      execution_result: { status: 'waiting_user' },
      ext_info: {
        step: 'agent_b_ask_user',
        iteration: iteration + 1,
        pending_key_fields: decision.data?.pendingKeyFields,
        available_solutions: decision.data?.availableSolutions,
      }
    }
  );

  // ========== 更新任务状态为 waiting_user ==========
  await db
    .update(agentSubTasks)
    .set({
      status: 'waiting_user',      // ← 关键状态！
      updatedAt: getCurrentBeijingTime(),
    })
    .where(eq(agentSubTasks.id, task.id));

  console.log('[SubtaskEngine] 任务已标记为等待用户处理: ' + task.id);
}
```

#### 交互记录详解（NEED_USER 会创建 2 条记录！）

| 记录序号 | interactType | interactNum | 内容 | 说明 |
|---------|-------------|------------|------|------|
| **记录 2** | `response` | `iteration` | 简化内容："需要用户介入" | Agent B 对执行 Agent 的回答 |
| **记录 3** | `request` | `iteration + 1` | **完整内容**：`pending_key_fields`、`available_solutions`、`prompt_message` | **关键！** Agent B 对用户的问题 |

#### 推进影响（重点！）

| 项目 | 影响 |
|-----|------|
| **任务状态** | `status` 从 `in_progress` → `waiting_user` ⏸️ |
| **流程继续** | ❌ **暂停** - `return` 结束 `executeCompleteWorkflow()` |
| **后续触发** | ✅ **需要用户反馈后重新触发** |
| **交互记录** | ✅ **创建 2 条记录**！<br>1. `response`（简化）<br>2. `request`（完整用户问题） |

#### 用户反馈后的重新触发（流程继续）

当用户提供反馈后，会通过其他 API 重新触发 `executeCompleteWorkflow()`：

```
用户在前端操作，提供反馈
    │
    ▼
API: /api/agents/user-decision 或类似接口
    │
    ▼
读取历史交互记录（agentSubTasksStepHistory）
    │
    ▼
恢复状态: mcpExecutionHistory、userInteractions、executorResult
    │
    ▼
再次调用 executeCompleteWorkflow(task)
    │
    ▼
Agent B 继续决策（从 historyRecords 恢复）
```

### 2.4 EXECUTE_MCP - 执行 MCP（唯一会继续的类型！）

#### 流程图（特殊！不会 return，而是 continue！）

```
Agent B 决策: type = "EXECUTE_MCP"
    │
    ▼
executeMcpWithRetry() ← 执行 MCP（最多 3 次尝试）
    │
    ├─ 尝试 1: 执行 MCP
    │   ├─ 成功 → return true
    │   └─ 失败 → 让 Agent B 重新决策，尝试 2
    │
    ├─ 尝试 2: 执行 MCP（Agent B 可能换方案）
    │   ├─ 成功 → return true
    │   └─ 失败 → 让 Agent B 重新决策，尝试 3
    │
    └─ 尝试 3: 执行 MCP
        ├─ 成功 → return true
        └─ 失败 → return false
    │
    ▼
switch 语句中判断 mcpSuccess
    │
    ├─ true (成功)
    │   └─ continue ← 关键！继续下一轮 while 循环！
    │           ↓
    │       Agent B 再次决策（判断是否完成）
    │
    └─ false (失败)
        └─ continue ← 也是 continue！
                ↓
            Agent B 再次决策（处理失败）
```

#### 代码逻辑（关键！不是 return，是 continue！）

```typescript
case 'EXECUTE_MCP':
  // 执行 MCP（支持多次尝试）
  const mcpSuccess = await this.executeMcpWithRetry(...);

  if (mcpSuccess) {
    // MCP 执行成功，继续下一轮决策让 Agent B 判断是否完成
    console.log('[SubtaskEngine] MCP执行成功，继续下一轮决策');
    continue;  // ← 关键！不是 return，是 continue ！
  } else {
    // MCP 多次尝试都失败，Agent B 会在下一轮决策中处理
    console.log('[SubtaskEngine] MCP多次尝试失败，继续下一轮决策');
    continue;  // ← 也是 continue ！
  }
```

**关键点对比**：

| 决策类型 | switch 中的处理 | 流程影响 |
|---------|----------------|---------|
| `COMPLETE` | `return;` | ❌ **结束** `executeCompleteWorkflow()` |
| `NEED_USER` | `return;` | ❌ **结束** `executeCompleteWorkflow()` |
| `FAILED` | `return;` | ❌ **结束** `executeCompleteWorkflow()` |
| `EXECUTE_MCP` | `continue;` | ✅ **继续** `while` 循环，Agent B 再次决策 |

#### executeMcpWithRetry() 详细步骤

**文件位置**: `src/lib/services/subtask-execution-engine.ts`  
**行号**: 1105-1355

```typescript
private async executeMcpWithRetry(
  task: typeof agentSubTasks.$inferSelect,
  initialDecision: AgentBDecision,
  capabilities: any[],
  mcpExecutionHistory: McpAttempt[],
  maxAttempts: number  // ← 通常是 3
): Promise&lt;boolean&gt; {
  let attemptCount = 0;
  let lastDecision = initialDecision;

  while (attemptCount &lt; maxAttempts) {
    attemptCount++;
    console.log('[SubtaskEngine] MCP尝试 ' + attemptCount + '/' + maxAttempts);

    // 执行 MCP
    const mcpResult = await this.executeCapabilityWithParams(...);

    // 构建 MCP 尝试记录，添加到历史
    const mcpAttempt: McpAttempt = { ... };
    mcpExecutionHistory.push(mcpAttempt);

    // 如果成功，返回 true
    if (mcpResult.success) {
      console.log('[SubtaskEngine] MCP执行成功');
      return true;  // ← 成功，返回 true
    }

    // 如果失败且还有尝试次数，让 Agent B 重新决策
    if (attemptCount &lt; maxAttempts) {
      console.log('[SubtaskEngine] MCP执行失败，请求Agent B重新决策');
      
      // 构建临时上下文用于重新决策
      const retryContext: ExecutionContext = { ... };
      
      // 调用 Agent B 重新决策！
      const retryDecision = await this.callAgentBWithDecision(...);
      
      if (retryDecision.type !== 'EXECUTE_MCP' || !retryDecision.data?.mcpParams) {
        console.log('[SubtaskEngine] Agent B决定不再继续MCP执行');
        return false;
      }
      
      lastDecision = retryDecision;  // ← 更新决策，继续循环
    }
  }

  return false;  // ← 所有尝试都失败，返回 false
}
```

#### 推进影响（唯一会继续的类型！）

| 项目 | 影响 |
|-----|------|
| **任务状态** | 保持 `in_progress` |
| **流程继续** | ✅ **继续** - `continue` 继续 `while` 循环 |
| **Agent B 决策** | ✅ **再次调用** `callAgentBWithDecision()` |
| **交互记录** | ✅ 每次 MCP 尝试都会记录到 `mcpExecutionHistory` |

## 六、总结（核心要点）

### 6.1 关键点 1 - 只有 EXECUTE_MCP 会继续

| 决策类型 | 是否 continue？ |
|---------|---------------|
| COMPLETE | ❌ No (return) |
| NEED_USER | ❌ No (return) |
| FAILED | ❌ No (return) |
| **EXECUTE_MCP** | ✅ **YES (continue)** |

### 6.2 关键点 2 - NEED_USER 会创建 2 条交互记录

| 记录 | interactType | 内容 |
|-----|-------------|------|
| 记录 2 | `response` | 简化："需要用户介入" |
| 记录 3 | `request` | **完整**：`pending_key_fields`、`available_solutions`、`prompt_message` |

### 6.3 关键点 3 - NEED_USER 会暂停流程，等待用户

- 状态变为 `waiting_user`
- `executeCompleteWorkflow()` 通过 `return` 结束
