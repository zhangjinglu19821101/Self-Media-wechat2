#### 1.2.1 `getPreviousStepResult()` 方法（第 1257 行）

```typescript
private getPreviousStepResult(
  allTasksInGroup: typeof agentSubTasks.$inferSelect[],
  currentOrderIndex: number
): any {
  if (currentOrderIndex &lt;= 1) {
    return null; // 第1步没有前序
  }
  
  // 查找 orderIndex-1 且 status=completed 的任务
  const previousOrderIndex = currentOrderIndex - 1;
  const previousTask = allTasksInGroup.find(
    t =&gt; t.orderIndex === previousOrderIndex &amp;&amp; t.status === 'completed'
  );
  
  // 返回前序任务的 executionResult
  if (previousTask &amp;&amp; previousTask.executionResult) {
    return JSON.parse(previousTask.executionResult);
  }
  
  return null;
}
```

## 四、当前问题分析

### 4.1 为什么链式传递没有生效？

**从数据看**：
- order_index=1 的 `execution_result` 有完整的文章内容 ✅
- order_index=2 的 `execution_result` 说"请提供需要审核的具体保险科普文章内容" ❌
- **说明链式传递逻辑没有生效**

**可能的原因**（需要进一步排查）：

1. **`allTasksInGroup` 参数不包含 order_index=1**
   - `getPreviousStepResult(allTasksInGroup, task.orderIndex)` 需要 `allTasksInGroup` 包含 order_index=1
   - 如果这个参数只包含 order_index=2，就找不到前序结果

2. **查询时 order_index=1 的状态不是 `completed`**
   - `getPreviousStepResult()` 要求 `t.status === 'completed'`
   - 虽然从最终数据看 order_index=1 是 `completed`
   - 但在 order_index=2 执行的那个时间点，order_index=1 的状态可能还不是 `completed`

3. **任务执行触发方式不对**
   - order_index=2 可能不是通过 `subtask-execution-engine` 的 `processGroup()` 方法触发的
   - 如果是通过其他方式触发的，可能没有传入 `allTasksInGroup` 参数

## 五、优化建议

### 5.1 短期优化（立即可以做）

1. **增加日志** ⭐⭐⭐⭐⭐
   - 在 `getPreviousStepResult()` 方法中增加详细日志
   - 输出 `allTasksInGroup` 的内容
   - 输出查询到的 `previousTask` 的状态
   - 这样可以快速定位问题

2. **增加调试接口** ⭐⭐⭐⭐
   - 提供一个接口，可以手动触发某个 order_index 的执行
   - 可以传入 `allTasksInGroup` 参数
   - 便于调试和测试

### 5.2 中期优化（1-2周）

1. **使用 JSONB 类型** ⭐⭐⭐
   - 将 `execution_result` 从 `text` 改为 `jsonb` 类型
   - PostgreSQL 的 JSONB 类型支持索引和查询
   - 性能更好，功能更强

2. **增加结果缓存** ⭐⭐
   - 对于已经完成的步骤，缓存其 execution_result
   - 减少数据库查询
   - 提升性能

### 5.3 长期优化（1个月）

1. **支持更复杂的依赖关系** ⭐⭐
   - 当前只支持 `order_index = N-1` 的依赖
   - 可以扩展为支持任意依赖（如 DAG）
   - 更灵活

2. **增加结果版本管理** ⭐
   - 支持结果的版本历史
   - 可以回退到任意版本
   - 更强大的错误恢复能力
