# 命名设计解释：wechatAddDraft vs addDraft

## 📊 完整命名关系图

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 实际导出的函数（底层实现）                                  │
│     export async function wechatAddDraft(params) { ... }        │
│                            ↑                                      │
│                            │  具体实现                             │
└────────────────────────────┼──────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 封装到对象中（暴露给外部调用）                               │
│     export const WechatMCPTools = {                              │
│       addDraft: wechatAddDraft,  ← 这里用简洁的名字！          │
│       uploadMedia: wechatUploadMedia,                            │
│       getAccounts: wechatGetAccounts,                            │
│       ...                                                         │
│     }                                                             │
│                            ↑                                      │
│                            │  简洁的方法名！                        │
└────────────────────────────┼──────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. capability_list 表存储（数据库中）                            │
│     id: 11                                                       │
│     tool_name: "wechat"                                           │
│     action_name: "addDraft"  ← 和对象中的方法名一致！           │
│                            ↑                                      │
│                            │  存的是简洁的名字！                     │
└────────────────────────────┼──────────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. 控制器调用时（运行时）                                        │
│     TOOL_MAP["wechat"] = WechatMCPTools                          │
│                                                            │
│     然后调用：                                                    │
│     const method = TOOL_MAP["wechat"]["addDraft"]               │
│                              ↑              ↑                     │
│                              │              │                     │
│                        tool_name    action_name                  │
│                                                            │
│     最后执行：                                                    │
│     await method(params)  ← 实际上就是 wechatAddDraft(params)！  │
└─────────────────────────────────────────────────────────────────┘
```

## 🔍 为什么这样设计？

### 理由 1：避免命名冲突

```typescript
// 底层 API 层（从微信官方 SDK 导入）
import { addDraft } from '@/lib/wechat-official-account/api';
//                       ↑
//                       这是微信的原始方法！

// 如果我们也叫 addDraft，就冲突了！❌
// export async function addDraft(params) { ... }  ← 冲突！

// 所以我们加个前缀，避免冲突！✅
export async function wechatAddDraft(params) { ... }
//                       ↑
//                    前缀，避免冲突！
```

---

### 理由 2：对外暴露时用简洁名字

```typescript
// 底层实现（带前缀，避免冲突）
export async function wechatAddDraft(params) { ... }
export async function wechatUploadMedia(params) { ... }
export async function wechatGetAccounts(params) { ... }

// 对外暴露的对象（去掉前缀，更简洁！）
export const WechatMCPTools = {
  addDraft: wechatAddDraft,      // ← 简洁！
  uploadMedia: wechatUploadMedia, // ← 简洁！
  getAccounts: wechatGetAccounts, // ← 简洁！
};

// 这样外部调用时更简洁！
// ✅ 好：WechatMCPTools.addDraft(params)
// ❌ 不好：WechatMCPTools.wechatAddDraft(params) （冗余！）
```

---

### 理由 3：capability_list 存的是对外暴露的名字

```typescript
// capability_list 存储的是：
// tool_name: "wechat"
// action_name: "addDraft"

// 因为我们对外暴露的就是这个名字！
export const WechatMCPTools = {
  addDraft: wechatAddDraft,  // ← 存的就是这个 key！
};

// 控制器调用时：
const tool = TOOL_MAP["wechat"];           // ← 找到 WechatMCPTools
const method = tool["addDraft"];            // ← 找到 wechatAddDraft
await method(params);                        // ← 执行！
```

### 1. 底层实现（带前缀）

```typescript
// 从微信 API 导入（也叫 addDraft！）
import { addDraft } from '@/lib/wechat-official-account/api';

export async function wechatAddDraft(params: WechatAddDraftParams) {

  // 实现...
```

```typescript

export const WechatMCPTools = {
```

### 4. 控制器调用

```typescript
// 伪代码：控制器如何调用
async function executeMCPCall(agentBOutput: any) {
  const { tool_name, action_name, params } = agentBOutput;
  
  // 1. 找到工具对象
  const tool = TOOL_MAP[tool_name];  // ← tool_name = "wechat"
  // 结果：tool = WechatMCPTools
  
  // 2. 找到方法
  const method = tool[action_name];   // ← action_name = "addDraft"
  // 结果：method = wechatAddDraft（底层函数！）
  
  // 3. 调用方法
  const result = await method(params);
  // 实际上就是：await wechatAddDraft(params)
  
  return result;
}
```

## 🎯 总结

| 位置 | 命名 | 原因 |
|------|------|------|
| **底层函数名** | `wechatAddDraft` | 避免与微信 API 的 `addDraft` 冲突 |
| **对象中的 key** | `addDraft` | 简洁、易读 |
| **capability_list.action_name** | `addDraft` | 和对象中的 key 保持一致 |
| **控制器调用时** | `TOOL_MAP["wechat"]["addDraft"]` | 直接用简洁名字调用 |
