# Agent T 提示词生效问题定位

## 问题描述

Agent T 返回的字段不稳定，例如 `needSplit` 和 `isNeedSplit` 混用，导致拆分逻辑不执行。

## 典型现象

**日志中的返回**：
```json
{ "needSplit": true, "splitReason": "任务包含两个独立步骤" }
```

**前端展示的返回**：
```json
{ "isNeedSplit": false, "splitReason": "任务包含两个独立步骤" }
```

**问题**：
- 模型理解了"需要拆分"（`splitReason` 有值）
- 但 `isNeedSplit` 返回了 `false`
- 而且 `suggestedSplitPoints` 是空数组

## 定位过程总结

### 第一步：查看 Agent T 原始返回日志

**关键日志位置**：

```bash
# 查看 Agent T 原始响应日志
tail -n 100 /app/work/logs/bypass/agent-t-raw-response.log
```

**日志格式**：
```json
{
  "timestamp": "2026-04-05T...",
  "taskId": "xxx",
  "orderIndex": 3,
  "response": { /* Agent T 原始返回 */ }
}
```

### 第二步：对比日志和前端展示

1. **如果日志中字段名和代码检查的字段名不一致** → 模型返回不稳定
2. **如果日志中字段值和代码判断的值不一致** → 可能是提示词问题

### 第三步：确认问题类型

#### 问题类型 1：模型返回字段名不稳定

**表现**：
- 日志显示：`needSplit: true`
- 前端显示：`isNeedSplit: false`
- 模型有时返回 `needSplit`，有时返回 `isNeedSplit`

**解决**：
```typescript
// 代码同时检查两个字段
const isAutoSplit = 
  agentTDecisionAny.needSplit === true ||
  agentTDecisionAny.isNeedSplit === true;
```

#### 问题类型 2：模型理解了但字段值写错

**表现**：
- `splitReason` 有值（模型理解了"需要拆分"）
- 但 `isNeedSplit` 是 `false`（写错了）
- `suggestedSplitPoints` 是空数组

**可能原因**：
1. 提示词格式复杂，模型在"选择分支"时出错
2. 模型不稳定，同一个提示词返回结果不一致
