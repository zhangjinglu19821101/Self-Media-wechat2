# 方式 2：运行时注册修改指南

## 📋 概述

使用方式 2（运行时注册），几乎不需要修改现有代码！

---

## ✅ 唯一需要做的修改

### 在 `src/app/layout.tsx` 中添加运行时注册

找到这个文件中的初始化部分，添加以下代码：

```typescript
// src/app/layout.tsx

import type { Metadata } from 'next';
import './globals.css';

// 🔥 在应用启动时初始化 WebSocket 服务器
import '@/lib/websocket-server';

// 🔥 在应用启动时启动 TS 定时任务、Agent B 巡检和定时任务调度器
import { TSScheduler } from '@/lib/services/ts-scheduler';
import { AgentBInspector } from '@/lib/services/agent-b-inspector';
import { startAllCronJobs } from '@/lib/cron';

// 🔥 新增：导入 ToolRegistry 和你的工具
import { toolRegistry } from '@/lib/mcp/tool-registry';
// 导入你的工具，比如：
// import { EmailMCPTools } from '@/lib/mcp/email-tools';

// 启动后台服务（仅在服务端运行）
if (typeof window === 'undefined') {
  // 启动 TS 定时任务（每 10 分钟检查一次）
  TSScheduler.start();
  console.log('✅ TS 定时任务已启动');

  // 启动 Agent B 巡检（每日 13:00）
  AgentBInspector.start();
  console.log('✅ Agent B 巡检服务已启动');

  // 启动定时任务调度器（轮巡、分发、监控、上报）
  startAllCronJobs();
  console.log('✅ 定时任务调度器已启动');

  // ============================================
  // 🔥 新增：在这里运行时注册你的 MCP 工具！
  // ============================================
  
  // 示例：注册邮件工具
  // toolRegistry.registerTool(
  //   'email',
  //   EmailMCPTools,
  //   '邮件相关工具：发送邮件'
  // );
  
  console.log('✅ MCP 工具运行时注册完成');
  console.log('✅ 可用工具:', toolRegistry.getAvailableTools());
}
```

## 📝 新增工具的完整步骤

### 步骤 1：创建工具实现文件

```typescript
// src/lib/mcp/email-tools.ts

export interface SendEmailParams {
  to: string;
  subject: string;
  body: string;
}

export const EmailMCPTools = {
  async sendEmail(params: SendEmailParams) {
    console.log('[Email Tools] 发送邮件:', params);
    // 实现你的逻辑
    return { success: true };
  }
};
```

### 步骤 2：在 `src/app/layout.tsx` 中注册

```typescript
// 在 src/app/layout.tsx 中添加：

import { EmailMCPTools } from '@/lib/mcp/email-tools';

// 然后在初始化部分添加：
toolRegistry.registerTool(
  'email',
  EmailMCPTools,
  '邮件相关工具：发送邮件'
);
```

### 步骤 3：在 `capability_list` 表中插入记录

```sql
INSERT INTO capability_list (
```

## ✅ 完成！

**就这么简单！**

- ✅ 不需要修改 `tool-registry.ts`
- ✅ 不需要修改 `generic-mcp-call.ts`
- ✅ 不需要修改 Agent B 的代码
- ✅ 只需要在 `src/app/layout.tsx` 中添加注册代码

---

## 🆚 与方式 1 的对比

| 维度 | 方式 1（在 tool-registry.ts） | 方式 2（运行时注册） |
|------|---------------------|-------------------|
| **需要修改的文件** | `tool-registry.ts` | `src/app/layout.tsx` |
| **需要重启吗？** | ✅ 需要 | ✅ 需要（但这是应用启动时注册） |
| **代码集中度** | ✅ 集中 | ⚠️ 分散 |
| **对 Agent B 影响** | ✅ 无影响 | ✅ 无影响 |

---

## 💡 提示

虽然方式 2 需要重启应用，但这是应用启动时的注册，不是每次新增工具都要重启！

如果你需要**真正的不重启动态注册**，可以：
1. 创建一个 API 路由来动态注册工具
2. 或者创建一个插件加载器，从文件系统动态加载

但对于大多数场景，在应用启动时运行时注册已经足够灵活了！
