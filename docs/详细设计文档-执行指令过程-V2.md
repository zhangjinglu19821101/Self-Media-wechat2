## 一、查询 agent_sub_tasks 表中待执行任务的条件

### 1.1 现有查询条件（保持不变）
```typescript
// 来自 src/lib/services/subtask-execution-engine.ts
private async getPendingTasks() {
  const today = new Date().toISOString().split('T')[0];

  return await db
    .select()
    .from(agentSubTasks)
    .where(
      and(
        // execution_date &lt;= 今天
        lte(agentSubTasks.executionDate, today),
        // status 为 pending 或 in_progress
        or(
          eq(agentSubTasks.status, 'pending'),
          eq(agentSubTasks.status, 'in_progress')
        )
      )
    )
    .orderBy(agentSubTasks.fromParentsExecutor, agentSubTasks.executionDate, agentSubTasks.orderIndex);
}
```

## 二、控制器重要的 4 个事情

### 2.1 通过状态机完成 agent_sub_tasks 表状态的更新

#### 2.1.1 状态机定义
```typescript
// 状态枚举
type SubTaskStatus =
  | 'pending'        // 待执行
  | 'in_progress'    // 执行中
  | 'need_support'   // 需要支持（Agent B 或用户）
  | 'waiting_agent_b'// 等待 Agent B 回应
  | 'waiting_user'   // 等待用户决策
  | 'completed'      // 已完成
  | 'failed';        // 失败

// 状态转换规则
const STATUS_TRANSITIONS = {
  'pending': ['in_progress', 'failed'],
  'in_progress': ['completed', 'failed', 'need_support'],
  'need_support': ['waiting_agent_b', 'waiting_user'],
  'waiting_agent_b': ['in_progress', 'need_support', 'waiting_user', 'completed', 'failed'],
  'waiting_user': ['in_progress', 'completed', 'failed'],
  'completed': [],
  'failed': [],
};
```

#### 2.1.2 状态更新触发机制
**状态更新完全来自于执行 Agent 的返回：**
- 执行 Agent 返回 `SUCCESS` → 状态更新为 `completed`
- 执行 Agent 返回 `FAILED` → 状态更新为 `failed`
- 执行 Agent 返回 `NEED_SUPPORT` → 状态更新为 `need_support`

---

### 2.2 调动 Agent B 与执行 Agent 的沟通与交互

#### 2.2.1 输入输出约定（已实现基础）
现有 `InteractContent` 类型（来自 `src/lib/types/interact-content.ts`）已经很完善，无需改动。

#### 2.2.2 一次交互的定义
**一次交互包括以下三种情况：**

1. **情况 1**：执行 Agent 反馈 + Agent B 应答
   - 执行 Agent → `interact_type = 'request'`
   - Agent B → `interact_type = 'response'`

2. **情况 2**：Agent B 提问 + 执行 Agent 反馈
   - Agent B → `interact_type = 'request'`
   - 执行 Agent → `interact_type = 'response'`

3. **情况 3**：用户提问 + 执行 Agent 反馈
   - 用户 → `interact_type = 'request'`
   - 执行 Agent → `interact_type = 'response'`

**上述三种情况，每完成一组即记录一次交互。**

#### 2.2.3 是否继续交互的决策
**完全由 Agent B 决策**，决策依据：
- 交互次数（建议最多 5 次）
- 执行 Agent 的反馈内容
- 问题是否已解决

---

### 2.3 Agent B 决策是否应该上报

#### 2.3.1 上报条件
**Agent B 在以下情况下务必上报：**
1. ✅ **交互次数达到上限（如 5 次）** - 必须上报
2. 其他情况 - Agent B 自行决策

#### 2.3.2 上报方式（已有弹框功能）
通过已有的弹框功能上报 Agent A：
- **弹框组件**：已有的 `Agent B` 问题监控页面
- **通知表**：`agent_notifications` 表
- **报告表**：`agent_reports` 表（已实现）

#### 2.3.3 上报前的登记
在 `agent_sub_tasks_step_history` 表中登记一个应答：
```typescript
// interact_type = 'request'
// consultant = 'agent_b'
// responder = 'agent_a'
// question = '问题总结'
// response = '建议上报 Agent A'
```

完成执行 Agent 上报问题的一个闭环。

---

### 2.4 用户通过 Agent A 对话框重新决策（新功能）

#### 2.4.1 功能描述
用户在 Agent A 对话框中收到上报信息后，重新做出下一步的决策。

#### 2.4.2 输入数据
将以下数据提交给执行 Agent：
1. **原始指令** - 最初的任务指令
2. **沟通历史记录** - `agent_sub_tasks_step_history` 表中的完整交互历史
3. **用户建议** - 用户在对话框中输入的建议/决策

#### 2.4.3 执行流程
1. 用户在 Agent A 对话框中查看上报信息
2. 用户输入决策/建议
3. 系统收集：原始指令 + 沟通历史 + 用户建议
4. 将上述数据提交给执行 Agent
5. 执行 Agent 重新执行任务

## 三、补充：执行 Agent 执行成功/失败后的事项

### 3.1 执行 Agent 执行成功，做哪些事项？

#### 3.1.1 状态更新
- 更新 `agent_sub_tasks.status` → `completed`
- 更新 `agent_sub_tasks.completedAt` → 当前时间
- 更新 `agent_sub_tasks.updatedAt` → 当前时间

#### 3.1.2 更新相关表
- 如果涉及 MCP，更新 MCP 相关状态
- 更新 `daily_tasks.completedSubTasks`（如有）
- 更新 `daily_tasks.completedSubTasksDescription`（如有）

#### 3.1.4 通知相关方（可选）
- 通知 Agent B 任务完成
- 通知 Agent A 任务完成（如需要）

---

### 3.2 执行 Agent 执行失败，做哪些事项？

#### 3.2.1 状态更新
- 更新 `agent_sub_tasks.status` → `failed`
- 更新 `agent_sub_tasks.updatedAt` → 当前时间

#### 3.2.2 记录失败原因
- 在 `agent_sub_tasks_step_history` 表中记录：
  - `interact_type = 'agent_response'`
  - `execution_result.status = 'failed'`
  - `execution_result.error_msg` → 失败原因

#### 3.2.3 判断是否需要支持
- 如果失败原因可解决，返回 `NEED_SUPPORT`
- 如果失败原因不可解决，保持 `failed` 状态

#### 3.2.4 通知相关方（可选）
- 通知 Agent B 任务失败
- 通知 Agent A 任务失败（如需要）

### A.2 MCP 多次尝试机制

单次 `interact_type=response` 支持多次 MCP 调用（最多3次）：

```typescript
interface McpAttempt {
  attemptId: string;            // 尝试唯一标识
  attemptNumber: number;        // 尝试序号（1, 2, 3）
  timestamp: Date;
  decision: {
    solutionNum: number;        // 方案编号
    toolName: string;           // 工具名称
    actionName: string;         // 动作名称
    reasoning: string;          // 选择理由
    strategy: 'initial' | 'retry' | 'switch_type' | 'degrade';
  };
  params: Record<string, any>;  // 调用参数
  result: {
    status: 'success' | 'failed' | 'partial';  // partial为预留状态，表示部分成功
    data?: any;                 // 成功时的数据
    error?: {                   // 失败时的错误
      code: string;
      message: string;
      type: string;
    };
    executionTime: number;
  };
  failureAnalysis?: {           // 失败分析
    isRetryable: boolean;
    failureType: 'temporary' | 'resource_unavailable';
    suggestedNextAction: 'retry_same' | 'switch_method';
  };
}
```

**重试策略：**
1. **首次尝试（initial）**：使用默认方案
2. **第二次尝试（retry）**：重试相同方案或切换 action 类型（如 searchEngine → simulateBrowser）
3. **第三次尝试（degrade）**：降级方案或最终失败

**failureType 说明：**
- **temporary**：临时错误（timeout、network），可重试
- **resource_unavailable**：资源不可用（permission、not_found 等），不可重试

#### C.3.1 错误类型定义

```typescript
type ErrorType =

  | 'temporary'         // 临时错误，可重试
  | 'resource_unavailable'; // 资源不可用，不可重试

interface ErrorClassification {
  errorType: ErrorType;
  failureType: FailureType;
  isRetryable: boolean;
  retryDelay: number;
```

#### C.3.2 错误分类规则

| 错误关键词/状态码 | ErrorType | FailureType | 是否可重试 | 建议操作 |
|------------------|-----------|-------------|-----------|---------|
| ECONNREFUSED, ENOTFOUND | network | temporary | ✅ | retry_same |
| ETIMEDOUT, timeout | timeout | temporary | ✅ | retry_same |
| 429 Too Many Requests | rate_limit | temporary | ✅ (延迟) | retry_same |
| 503 Service Unavailable | service_error | temporary | ✅ | retry_same |
| 403 Forbidden | permission | resource_unavailable | ❌ | switch_method |
| 404 Not Found | not_found | resource_unavailable | ❌ | switch_method |
| 400 Bad Request | invalid_params | resource_unavailable | ❌ | abort |
| 401 Unauthorized | permission | resource_unavailable | ❌ | abort |

#### C.3.3 错误分类方法实现

```typescript
/**
 * 根据错误信息分类错误类型
 */
private classifyErrorType(error: any): ErrorType {
  const errorMessage = (error?.message || String(error)).toLowerCase();
  const errorCode = error?.code || error?.statusCode;
  
  // 网络错误
  if (['econnrefused', 'enotfound', 'econnreset', 'socket hang up'].some(k => 
    errorMessage.includes(k))) {
    return 'network';
  }
  
  // 超时错误
  if (['timeout', 'etimedout', 'request timeout'].some(k => 
    errorMessage.includes(k))) {
    return 'timeout';
  }
  
  // 根据HTTP状态码判断
  if (errorCode === 429) return 'rate_limit';
  if (errorCode === 403) return 'permission';
  if (errorCode === 404) return 'not_found';
  if (errorCode === 400) return 'invalid_params';
  if (errorCode >= 500) return 'service_error';
  
  return 'unknown';
}

/**
 * 判断错误是否可重试
 */
private isRetryableError(errorType: ErrorType): boolean {
  const retryableTypes: ErrorType[] = ['network', 'timeout', 'rate_limit', 'service_error'];
  return retryableTypes.includes(errorType);
}

/**
 * 获取失败类型
 */
private getFailureType(errorType: ErrorType): FailureType {
  const temporaryTypes: ErrorType[] = ['network', 'timeout', 'rate_limit', 'service_error'];
  return temporaryTypes.includes(errorType) ? 'temporary' : 'resource_unavailable';
}

/**
 * 获取重试延迟时间
 */
private getRetryDelay(errorType: ErrorType, attemptNumber: number): number {
  const delays = RETRY_DELAYS[errorType === 'network' ? 'network' : 'temporary'];
  return delays[Math.min(attemptNumber - 1, delays.length - 1)] || 4000;
}
```

### C.4 带重试的MCP执行实现

```typescript
/**
 * 带重试机制的MCP执行
 * 单次迭代中最多尝试3次，支持自动重试和策略切换
 */
private async executeMcpWithRetry(
  task: typeof agentSubTasks.$inferSelect,
  initialDecision: AgentBDecision,
  capabilities: any[],
  mcpExecutionHistory: McpAttempt[],
  maxAttempts: number = MAX_MCP_ATTEMPTS_PER_ITERATION
): Promise<boolean> {
  let currentAttempt = 0;
  let currentDecision = initialDecision;
  
  while (currentAttempt < maxAttempts) {
    currentAttempt++;
    
    try {
      // 执行MCP调用
      const mcpResult = await this.executeMcp(
        task, 
        currentDecision.data?.mcpParams!,
        capabilities
      );
      
      // 构建尝试记录
      const attempt: McpAttempt = {
        attemptId: `mcp-${Date.now()}-${currentAttempt}`,
        attemptNumber: currentAttempt,
        timestamp: new Date(),
        decision: {
          solutionNum: currentDecision.data?.solutionNum || 1,
          toolName: currentDecision.data?.mcpParams?.toolName || '',
          actionName: currentDecision.data?.mcpParams?.action || '',
          reasoning: currentDecision.reasoning,
          strategy: currentAttempt === 1 ? 'initial' : 
                    currentAttempt === 2 ? 'retry' : 'degrade'
        },
        params: currentDecision.data?.mcpParams || {},
        result: {
          status: mcpResult.success ? 'success' : 'failed',
          data: mcpResult.success ? mcpResult.data : undefined,
          error: !mcpResult.success ? {
            code: mcpResult.error?.code || 'UNKNOWN',
            message: mcpResult.error?.message || 'Unknown error',
            type: this.classifyErrorType(mcpResult.error)
          } : undefined,
          executionTime: mcpResult.executionTime || 0
        }
      };
      
      // 如果成功，记录并返回
      if (mcpResult.success) {
        mcpExecutionHistory.push(attempt);
        return true;
      }
      
      // 失败时进行错误分析
      const errorType = this.classifyErrorType(mcpResult.error);
      const isRetryable = this.isRetryableError(errorType);
      const failureType = this.getFailureType(errorType);
      
      attempt.failureAnalysis = {
        isRetryable,
        failureType,
        suggestedNextAction: isRetryable ? 'retry_same' : 'switch_method'
      };
      
      mcpExecutionHistory.push(attempt);
      
      // 如果是最后一次尝试或不可重试，返回失败
      if (currentAttempt >= maxAttempts || !isRetryable) {
        return false;
      }
      
      // 等待后重试
      const delay = this.getRetryDelay(errorType, currentAttempt);
      await new Promise(resolve => setTimeout(resolve, delay));
      
      // 请求Agent B生成新的参数（如果是可重试错误）
      if (isRetryable && currentAttempt < maxAttempts) {
        const newDecision = await this.callAgentBForRetry(
          task,
          mcpExecutionHistory,
          errorType
        );
        if (newDecision) {
          currentDecision = newDecision;
        }
      }
      
    } catch (error) {
      // 捕获意外错误
      const attempt: McpAttempt = {
        attemptId: `mcp-${Date.now()}-${currentAttempt}`,
        attemptNumber: currentAttempt,
        timestamp: new Date(),
        decision: {
          solutionNum: currentDecision.data?.solutionNum || 1,
          toolName: currentDecision.data?.mcpParams?.toolName || '',
          actionName: currentDecision.data?.mcpParams?.action || '',
          reasoning: currentDecision.reasoning,
          strategy: 'initial'
        },
        params: currentDecision.data?.mcpParams || {},
        result: {
          status: 'failed',
          error: {
            code: 'EXECUTION_EXCEPTION',
            message: error instanceof Error ? error.message : String(error),
            type: 'unknown'
          },
          executionTime: 0
        },
        failureAnalysis: {
          isRetryable: false,
          failureType: 'resource_unavailable',
          suggestedNextAction: 'switch_method'
        }
      };
      
      mcpExecutionHistory.push(attempt);
      
      if (currentAttempt >= maxAttempts) {
        return false;
      }
    }
  }
  
  return false;
}
```

### C.5 决策类型与处理逻辑

| 决策类型 | 触发条件 | 处理逻辑 | 交互历史记录 |
|---------|---------|---------|-------------|
| **EXECUTE_MCP** | 需要执行MCP能力 | 1. 调用 `executeMcpWithRetry`<br>2. 最多3次尝试<br>3. 成功后继续下一轮迭代 | 记录 request + response |
| **COMPLETE** | 任务已完成 | 1. 更新任务状态为 `completed`<br>2. 记录最终结果<br>3. 结束执行循环 | 记录最终 response |
| **NEED_USER** | 需要用户确认 | 1. 更新任务状态为 `waiting_user`<br>2. 记录待确认字段<br>3. 暂停等待用户输入 | 记录 response + 等待 request |
| **FAILED** | 执行失败且无法自动恢复 | 1. 更新任务状态为 `failed`<br>2. 记录失败原因<br>3. 可选：上报Agent A | 记录失败 response |

```typescript
private async handleMaxIterationsExceeded(
  task: typeof agentSubTasks.$inferSelect,
  commandResultId: string,
  currentStepNo: number,
  interactNum: number,
  executionContext: ExecutionContext
): Promise<void> {
  // 1. 更新任务状态为需要支持
  await db.update(agentSubTasks)
    .set({ status: 'need_support' })
    .where(eq(agentSubTasks.id, task.id));
  
  // 2. 记录交互历史
  await this.createInteractionStep(
    commandResultId,
    currentStepNo,
    'response',
    interactNum,
    'agent B',
    {
      interact_type: 'response',
      consultant: 'insurance-d',
      responder: 'agent B',
      question: executionContext.lastRequest,
      response: {
        decision: {
          type: 'FAILED',
          reason_code: 'MAX_ITERATIONS_EXCEEDED',
          reasoning: `已达到最大迭代次数（${MAX_ITERATIONS}次），无法自动完成任务`,
          final_conclusion: '需要上报Agent A或人工介入处理',
          context: {
            execution_summary: executionContext.executionSummary,
            risk_level: 'high',
            suggested_action: '上报Agent A进行人工决策'
          }
        },
        mcp_attempts: executionContext.mcpAttempts,
        execution_summary: {
          total_mcp_attempts: executionContext.mcpAttempts.length,
          successful_mcp_attempts: executionContext.mcpAttempts.filter(a => a.result.status === 'success').length,
          failed_mcp_attempts: executionContext.mcpAttempts.filter(a => a.result.status === 'failed').length,
          total_user_interactions: executionContext.userInteractions.length,
          start_time: executionContext.startTime,
          end_time: new Date().toISOString(),
          total_duration: Date.now() - new Date(executionContext.startTime).getTime()
        }
      },
      execution_result: {
        status: 'need_support',
        error_msg: `Max iterations (${MAX_ITERATIONS}) exceeded`
      },
      ext_info: {
        step: 'max_iterations_exceeded',
        iteration: MAX_ITERATIONS,
        max_iterations_reached: true
      }
    }
  );
  
  // 3. 上报Agent A（通过已有弹框功能）
  await this.reportToAgentA(task, executionContext);
}
```
