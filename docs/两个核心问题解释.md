# 两个核心问题解释

---

## 问题 1：TOOL_MAP 和 WechatMCPTools 是什么意思？

### 1.1 先看一个生活中的类比

```
想象你有一个工具箱：

TOOL_MAP = {
  "厨房": 厨房工具箱,
  "书房": 书房工具箱,
}

厨房工具箱 = {
  "切菜": 菜刀,
  "炒菜": 锅铲,
  "煮饭": 电饭煲,
}

书房工具箱 = {
  "写字": 钢笔,
  "看书": 眼镜,
  "打印": 打印机,
}
```

**现在你要"切菜"：**
```
1. 先从 TOOL_MAP 找 "厨房" → 得到厨房工具箱
2. 再从厨房工具箱找 "切菜" → 得到菜刀
3. 用菜刀切菜！
```

---

### 1.2 回到代码，是一个意思！

```typescript
// 👇 这个是：工具箱的收纳柜
const TOOL_MAP = {
  "search": SearchMCPTools,    // ← "search" 是一个工具箱
  "wechat": WechatMCPTools,    // ← "wechat" 是另一个工具箱
};

// 👇 这个是：微信工具箱里面有什么工具
export const WechatMCPTools = {
  "addDraft": wechatAddDraft,      // ← "addDraft" 是一个工具（函数）
  "uploadMedia": wechatUploadMedia, // ← "uploadMedia" 是另一个工具（函数）
  "getAccounts": wechatGetAccounts, // ← "getAccounts" 是另一个工具（函数）
};
```

**现在你要调用"addDraft"：**
```typescript
// 1. 先从 TOOL_MAP 找 "wechat" → 得到 WechatMCPTools（微信工具箱）
const tool = TOOL_MAP["wechat"];

// 2. 再从 WechatMCPTools 找 "addDraft" → 得到 wechatAddDraft（函数）
const method = tool["addDraft"];

// 3. 调用这个函数！
const result = await method(params);
```

---

### 1.3 为什么要这样设计？

**如果不这样设计，你要写很多 if-else：**

```typescript
// ❌ 不好的设计：写一堆 if-else
async function badDesign(tool_name, action_name, params) {
  if (tool_name === "wechat") {
    if (action_name === "addDraft") {
      return await wechatAddDraft(params);
    } else if (action_name === "uploadMedia") {
      return await wechatUploadMedia(params);
    } else if (action_name === "getAccounts") {
      return await wechatGetAccounts(params);
    }
  } else if (tool_name === "search") {
    if (action_name === "webSearch") {
      return await searchWebSearch(params);
    }
  }
  // ... 继续写一堆 if-else
}

// ✅ 好的设计：用对象映射
async function goodDesign(tool_name, action_name, params) {
  const tool = TOOL_MAP[tool_name];        // ← 一行代码找到工具箱
  const method = tool[action_name];         // ← 一行代码找到工具
  return await method(params);               // ← 一行代码调用！
}
```

## 问题 2：为什么要在 capability_list 中存那么多字段？

### 2.1 先看 capability_list 有哪些字段

```sql
-- capability_list 表的关键字段
id,
capability_type,
function_desc,          -- ← 功能描述
tool_name,              -- ← 工具名（如 "wechat"）
action_name,            -- ← 动作名（如 "addDraft"）
param_examples,         -- ← 参数示例
param_template,         -- ← 参数模板
interface_schema,       -- ← 参数规范
agent_response_spec,    -- ← Agent B 返回规范
```

---

### 2.2 每个字段的作用（用一个故事解释）

**想象你要教一个机器人（Agent B）帮你做事情：**

```
你："机器人，帮我上传一篇微信公众号文章！"

机器人："我不会啊，你得告诉我："
  1. 用什么工具？（tool_name）
  2. 用工具的哪个功能？（action_name）
  3. 参数长什么样？（interface_schema）
  4. 能给我个例子吗？（param_examples）
  5. 能给我个模板填空吗？（param_template）
  6. 我做完后怎么告诉你结果？（agent_response_spec）

你："好的，我都写到 capability_list 表里了，你自己去查！"

机器人去查 capability_list 表：
  - tool_name = "wechat" → 知道用微信工具
  - action_name = "addDraft" → 知道用添加草稿功能
  - param_examples → 看到了参数示例
  - param_template → 拿到了填空模板
  - interface_schema → 知道了参数规范
  - agent_response_spec → 知道了怎么返回结果

机器人："好的，我会了！"
```

---

### 2.3 每个字段的具体作用

| 字段 | 作用 | 给谁用？ |
|------|------|---------|
| `tool_name` | 用什么工具？（"wechat" / "search"） | 控制器 |
| `action_name` | 用工具的哪个功能？（"addDraft"） | 控制器 |
| `function_desc` | 这个能力是做什么的？（给人看的描述） | Agent B + 人类 |
| `param_examples` | 参数长什么样？给个例子 | Agent B |
| `param_template` | 给我个模板，我直接填空 | Agent B |
| `interface_schema` | 参数的严格规范（类型、必填等） | Agent B + 验证 |
| `agent_response_spec` | 我做完后怎么返回结果？ | Agent B |

---

### 2.4 如果没有这些字段会怎么样？

**场景 1：没有 param_examples**
```
机器人："参数怎么拼啊？给个例子呗？"
你："自己猜！"
机器人："我猜不出来..." ❌
```

**场景 2：没有 interface_schema**
```
机器人："这个字段是必填的吗？类型是字符串还是数字？"
你："自己看文档！"
机器人："我找不到文档..." ❌
```

**场景 3：没有 agent_response_spec**
```
机器人："我做完了，怎么告诉你结果？"
你："随便！"
机器人返回了一堆乱码... ❌
```

**场景 4：有了这些字段 ✅**
```
机器人："我都查到了！"
  - param_examples → 有例子
  - param_template → 有模板
  - interface_schema → 有规范
  - agent_response_spec → 知道怎么返回
机器人："放心交给我！" ✅
```

## 🎯 总结

### 问题 1 总结：TOOL_MAP 和 WechatMCPTools 是什么？

```typescript
// TOOL_MAP = 工具箱的收纳柜
const TOOL_MAP = {
  "wechat": WechatMCPTools,    // ← 微信工具箱
  "search": SearchMCPTools,     // ← 搜索工具箱
};

// WechatMCPTools = 微信工具箱里面有什么
export const WechatMCPTools = {
  "addDraft": wechatAddDraft,      // ← 菜刀（切菜用）
  "uploadMedia": wechatUploadMedia, // ← 锅铲（炒菜用）
};

// 使用方式
const tool = TOOL_MAP["wechat"];           // ← 拿出微信工具箱
const method = tool["addDraft"];            // ← 拿出菜刀
await method(params);                        // ← 用菜刀切菜！
```

### 问题 2 总结：为什么要那么多字段？

**因为 Agent B 是个机器人，你得把所有信息都告诉它！**

| 字段 | 告诉 Agent B 什么？ |
|------|-------------------|
| `tool_name` | 用什么工具？ |
| `action_name` | 用工具的哪个功能？ |
| `function_desc` | 这个能力是做什么的？ |
| `param_examples` | 参数长什么样？给个例子 |
| `param_template` | 给我个模板填空 |
| `interface_schema` | 参数的严格规范 |
| `agent_response_spec` | 怎么返回结果 |

**如果少了任何一个字段，Agent B 就会说："我不会！"** 😅
