## 2026-03-31 - order_index=2 任务卡在 pending 状态问题

**问题**: order_index=2（合规审核）任务卡在 `pending` 状态，数据库显示 `status=pending`，但 `metadata` 显示 `autoCompletedDueToExhausted=true`，状态不一致。

**根本原因**: `EXECUTE_MCP` 决策处理把状态设置为 `pending`：
```typescript
status: 'pending',  // 回到 pending，重新进入 Agent B 评审
```

但 `checkAgentBReviewTaskBeforeExecution` 的防重检查只接受 `pre_need_support` 或 `pre_completed` 状态：
```typescript
const validStatesForReview = ['pre_need_support', 'pre_completed'];
```

**影响**: 状态设置为 `pending` 后，防重检查失败，任务永远卡在 `pending`！

**修复**:
- **文件**: `src/lib/services/subtask-execution-engine.ts`
- **位置**: 第 3921 行
- **改动**: `status: 'pending'` → `status: 'pre_completed'`

**语义说明**:
- `pre_need_support`：执行 Agent 认为**需要帮助/支持**
- `pre_completed`：执行 Agent 认为任务**可以完成**
- MCP 执行成功后，执行 Agent 应该基于 MCP 结果完成任务，所以用 `pre_completed`

**说明**:
- `REEXECUTE_EXECUTOR` 设置 `pending` 是**正确**的（让任务重新进入执行阶段）
- `EXECUTE_MCP` 设置 `pending` 是**错误**的（应该让任务重新进入评审阶段 → `pre_completed`）
