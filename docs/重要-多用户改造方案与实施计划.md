## 一、修订说明（v1 → v2）

| 修订项 | v1 方案 | v2 修订 | 原因 |
|--------|---------|---------|------|
| 用户模型 | 企业作为 user.role | Workspace 独立表 | 解决身份二义性、资源归属清晰 |
| 配额系统 | Phase 2 实现 | **降级为「后续讨论」** | 用户自选 LLM 模型、费用自理，我们只提供工具 |
| 权限模型 | 20+ Permission 枚举 | 简化为 3 层角色 | 降低维护成本，够用即可 |
| 发布流程 | 独立 Phase 4 | 与任务完成流程打通 | 避免脱节 |
| 风格模板兼容 | 未深入讨论 | **新增专门章节** | 这是系统核心资产，必须平滑迁移 |

## 二、项目背景与目标

### 2.1 背景

当前系统为**单用户架构**，所有数据无租户隔离、无认证鉴权。核心能力链路：

```
用户输入指令
    ↓
Agent B 拆解任务（agent_sub_tasks）
    ↓
insurance-d 创作（使用 accountId → templateId → style_rules）
    ↓
Agent B 评审
    ↓
【当前终点：文章生成完成】
    ↓
【新增：多平台发布】
```

### 2.2 核心目标

| 目标 | 说明 | 优先级 |
|------|------|--------|
| **Workspace 多租户** | 个人/企业团队共享资源，数据严格隔离 | P0 |
| **用户认证登录** | 注册/登录/Session 管理 | P0 |
| **数据租户隔离** | 所有资源按 Workspace 过滤 | P0 |
| **多平台发布** | 创作完成后一键发布到各平台 | P0 |
| **风格模板兼容** | 现有 accountId→templateId→style_rules 链路在多用户下正常工作 | P0 |
| **基础权限控制** | Owner/Admin/Editor/Viewer 四级角色 | P1 |

### 2.3 不在本次范围（后续讨论）

| 项目 | 说明 | 讨论时机 |
|------|------|----------|
| 配额/限流系统 | 用户自选 LLM 模型，费用自理 | 产品侧确定商业模式后 |
| 支付体系 | 取决于配额策略 | 配额确定后 |
| 审计日志 | 企业合规需求 | 有企业客户后 |
| LLM 模型选择器 | 让用户在 Coze 平台选择模型 | 技术可行性评估后 |

### 2.4 约束条件

- ✅ 系统未上线，**无需历史数据迁移**
- ✅ 现有表结构可自由修改
- ⚠️ **风格模板系统是核心资产**，改造必须保证现有链路不中断：
  - `platform_accounts` → `account_style_configs` → `style_templates` → `style_assets`
  - 执行引擎 `getTemplateIdForTask()` 的 accountId → templateId 查询链路
- ⚠️ 改造期间保持现有功能可用

### 3.2 风格模板系统关键链路（必须保持兼容）

这是系统最核心的数据流，**任何改造都不能破坏这条链路**：

```
┌─────────────────────────────────────────────────────────────────┐
│                    风格规则加载链路（执行时）                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ① 用户在 full-home 选择"发布账号"                               │
│     ↓                                                           │
│  ② accountId 存入 agent_sub_tasks.metadata.accountId             │
│     ↓                                                           │
│  ③ SubtaskExecutionEngine.getTemplateIdForTask(task):           │
│     ├─ 读取 task.metadata.accountId                             │
│     ├─ styleTemplateService.getAccountPlatform(accountId)       │
│     │   → 查 platform_accounts 表获取平台类型                    │
│     ├─ styleTemplateService.getTemplateIdByAccount(accountId)   │
│     │   → 查 account_style_configs 表获取绑定模板ID              │
│     │   → 若未绑定 → getDefaultTemplate(userId, platform)       │
│     └─ 返回 templateId                                         │
│     ↓                                                           │
│  ④ PromptAssemblerService:                                      │
│     ├─ digitalAssetService.getStyleRules(templateId)            │
│     │   → 查 style_assets 表 WHERE templateId = ?               │
│     ├─ digitalAssetService.getUserExclusiveRules(userId)        │
│     └─ 将规则注入 insurance-d 提示词                            │
│     ↓                                                           │
│  ⑤ insurance-d 使用风格规则生成文章                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

涉及的表关系：
  platform_accounts.id ──→ account_style_configs.accountId
  style_templates.id   ──→ account_style_configs.templateId
  style_assets.templateId ──→ style_templates.id
```

## 四、改造方案详解

### 4.1 Phase 1：Workspace + 认证系统（Week 1-2.5）

#### 4.1.1 核心设计理念：Workspace 模式

借鉴 Notion/Figma/Slack 的成熟模式：

```
┌─────────────────────────────────────────────────────────────┐
│                      用户 (Account)                          │
│                                                             │
│  每个自然人一个 Account，用邮箱登录                           │
│  Account 本身不存储业务数据                                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          ↓               ↓               ↓
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│ Personal WS  │  │ Enterprise A │  │ Enterprise B │
│ (个人工作空间) │  │ (企业A空间)  │  │ (企业B空间)  │
│              │  │              │  │              │
│ • 默认创建    │  │ • 邀请加入    │  │ • 邀请加入    │
│ • 个人数据    │  │ • 团队数据    │  │ • 团队数据    │
└──────────────┘  └──────────────┘  └──────────────┘
     ↑                   ↑                   ↑
  所有资源归           所有资源归           所有资源归
  此 Workspace         此 Workspace         此 Workspace
```

**关键原则**：
- Account 是身份凭证（登录用）
- Workspace 是资源容器（所有业务数据归属这里）
- 一个 Account 可以属于多个 Workspace（个人 + 多个企业）
- 资源（任务/素材/模板/账号）永远归属于 Workspace，不属于个人

#### 4.1.2 数据库 Schema

```sql
-- ============================================
-- 表 1：accounts（账户表 — 自然人身份）
-- ============================================
CREATE TABLE accounts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    
    -- 登录凭证
    email TEXT UNIQUE NOT NULL,
    email_verified BOOLEAN DEFAULT FALSE,
    password_hash TEXT NOT NULL,
    
    -- 基本信息
    name TEXT NOT NULL,
    avatar_url TEXT,
    phone TEXT,
    
    -- 安全
    failed_login_attempts INTEGER DEFAULT 0,
    locked_until TIMESTAMPTZ,
    
    -- 偏好
    timezone TEXT DEFAULT 'Asia/Shanghai',
    locale TEXT DEFAULT 'zh-CN',
    
    -- 时间戳
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    last_login_at TIMESTAMPTZ,
    last_login_ip TEXT
);

CREATE UNIQUE INDEX idx_accounts_email ON accounts(email);

-- ============================================
-- 表 2：workspaces（工作空间 — 资源容器）
-- ============================================
CREATE TABLE workspaces (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    
    -- 基本信息
    name TEXT NOT NULL,                    -- "我的工作空间" / "XX公司"
    slug TEXT UNIQUE NOT NULL,             -- URL 友好标识
    
    -- 类型
    type TEXT NOT NULL DEFAULT 'personal', -- personal / enterprise
    
    -- 所有者
    owner_account_id UUID NOT NULL REFERENCES accounts(id) ON DELETE CASCADE,
    
    -- 企业信息（type=enterprise 时填写）
    company_name TEXT,
    
    -- 时间戳
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_workspaces_owner ON workspaces(owner_account_id);
CREATE INDEX idx_workspaces_type ON workspaces(type);

-- ============================================
-- 表 3：workspace_members（成员关系）
-- ============================================
CREATE TABLE workspace_members (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    
    workspace_id UUID NOT NULL REFERENCES workspaces(id) ON DELETE CASCADE,
    account_id UUID NOT NULL REFERENCES accounts(id) ON DELETE CASCADE,
    
    -- 角色（简化为 4 级）
    role TEXT NOT NULL DEFAULT 'viewer',  -- owner / admin / editor / viewer
    
    -- 状态
    status TEXT NOT NULL DEFAULT 'active',  -- active / pending_invitation / removed / left
    
    -- 邀请信息
    invited_by UUID REFERENCES accounts(id),
    invited_at TIMESTAMPTZ DEFAULT NOW(),
    joined_at TIMESTAMPTZ,
    
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 每个账户在每个工作空间只有一条记录
CREATE UNIQUE INDEX idx_workspace_members_unique 
    ON workspace_members(workspace_id, account_id);
CREATE INDEX idx_workspace_members_account ON workspace_members(account_id);

-- ============================================
-- 表 4：account_sessions（会话表）
-- ============================================
CREATE TABLE account_sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    account_id UUID NOT NULL REFERENCES accounts(id) ON DELETE CASCADE,
    token TEXT NOT NULL UNIQUE,
    device_info JSONB,
    ip_address INET,
    expires_at TIMESTAMPTZ NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_sessions_account ON account_sessions(account_id);
CREATE INDEX idx_sessions_token ON account_sessions(token);
```

#### 4.1.3 与现有系统的映射关系

**这是最关键的兼容性设计**。现有系统中 `userId` 的语义需要重新定义为 `workspaceId`：

```
现有字段              新含义              迁移方式
─────────────────────────────────────────────────────
style_templates.user_id  →  workspace_id    ALTER RENAME
platform_accounts.user_id → workspace_id   ALTER RENAME  
account_style_configs.user_id → workspace_id  ALTER RENAME
style_assets.user_id     →  workspace_id    ALTER RENAME
material_library.user_id →  workspace_id   ALTER RENAME
... (其余 7 张已有 user_id 的表同理)

agent_tasks             →  新增 workspace_id 字段
daily_task              →  新增 workspace_id 字段
agent_sub_tasks         →  新增 workspace_id 字段
... (其余 11 张缺字段的表)
```

**为什么可以安全重命名**：
- 系统未上线，无生产数据
- 开发环境数据可在迁移时清理或批量更新
- `userId` 和 `workspace_id` 都是 UUID 类型，无需改类型

```typescript
/**
 * 获取账户可访问的所有 workspace ID
 * - Personal workspace: 自己的
 * - Enterprise workspace: 自己是 active 成员的
 */
export async function getAccessibleWorkspaceIds(accountId: string): Promise<string[]> {
  const { workspaces, workspaceMembers } = await import('@/lib/db/schema/auth');
  
  // 1. 自己拥有的 workspace
  const owned = await db.select({ id: workspaces.id })
    .from(workspaces)
    .where(eq(workspaces.ownerAccountId, accountId));
  
  // 2. 自己参与的 workspace（active 成员）
  const memberOf = await db.select({ workspaceId: workspaceMembers.workspaceId })
    .from(workspaceMembers)
    .where(
      and(
        eq(workspaceMembers.accountId, accountId),
        eq(workspaceMembers.status, 'active')
      )
    );
  
  const ids = [
    ...owned.map(w => w.id),
    ...memberOf.map(m => m.workspaceId),
  ];
  
  // 去重
  return [...new Set(ids)];
}
```

```typescript
    .from(agentTasks)
    .where(eq(agentTasks.workspaceId, workspaceId));
  
  return NextResponse.json({ success: true, data: tasks });
```

#### 4.3.1 简化的 4 级角色模型

```typescript
// src/lib/auth/roles.ts

/** 角色枚举（简化版） */
export enum WorkspaceRole {
  OWNER = 'owner',      // 全部权限 + Workspace 设置 + 成员管理
  ADMIN = 'admin',      // 全部数据权限（除删除 Workspace 外）
  EDITOR = 'editor',    // 可读写数据（不能删他人资源、不能管理成员）
  VIEWER = 'viewer',    // 只读（仅查看数据和报表）
}

/** 权限判断矩阵（硬编码，简单清晰） */
export function canDo(role: WorkspaceRole, action: Action, resourceOwner?: string, actorId?: string): boolean {
  switch (action) {
    case 'workspace:manage':
    case 'member:invite':
    case 'member:remove':
    case 'member:change_role':
      return role === WorkspaceRole.OWNER;
      
    case 'resource:delete_any':
      return role === WorkspaceRole.OWNER || role === WorkspaceRole.ADMIN;
      
    case 'resource:delete_own':
      if (role === WorkspaceRole.VIEWER) return false;
      return resourceOwner === actorId;  // 只能删自己的
      
    case 'resource:create':
    case 'resource:update_own':
      if (role === WorkspaceRole.VIEWER) return false;
      return true;
      
    case 'resource:read':
      return true;  // 所有角色都可读
      
    case 'publish:create':
    case 'publish:cancel':
      if (role === WorkspaceRole.VIEWER) return false;
      return true;
      
    default:
      return false;
  }
}

export type Action =
  | 'workspace:manage'
  | 'member:invite' | 'member:remove' | 'member:change_role'
  | 'resource:create' | 'resource:read' | 'resource:update_own' | 'resource:delete_own' | 'resource:delete_any'
  | 'publish:create' | 'publish:cancel';
```

#### 4.3.2 资源可见性策略

明确每种资源的归属和可见性：

| 资源 | 归属 | 可见范围 | 说明 |
|------|------|----------|------|
| **风格模板** (`style_templates`) | Workspace | Workspace 全员 | 企业资产，全员可用 |
| **平台账号** (`platform_accounts`) | Workspace | Workspace 全员 | 企业资产，但发布需审批 |
| **素材库** (`material_library`) | Workspace | Workspace 全员 | 企业资产 |
| **风格规则** (`style_assets`) | Workspace（通过模板） | Workspace 全员 | 跟随模板可见 |
| **创作任务** (`agent_tasks`) | Workspace | Workspace 全员 | 团队协作查看 |
| **子任务** (`agent_sub_tasks`) | Workspace（跟随父任务） | Workspace 全员 | 跟随父任务可见 |
| **执行历史** (`step_history`) | Workspace（跟随子任务） | Workspace 全员 | 跟随子任务可见 |
| **发布记录** (`publish_records`) | Workspace | Workspace 全员 | 团队可见发布状态 |
| **核心锚点** (`core_anchor_assets`) | Workspace | Workspace 全员 | 企业资产 |

**结论：本系统采用「Workspace 全员可见」策略**，不做私有资源。

理由：
1. 自媒体场景本质是团队协作产出内容
2. 隐藏资源会增加复杂度，收益有限
3. 如未来需要，可通过简单的 `visibility` 字段扩展

#### 4.3.3 权限装饰器

```typescript
// src/lib/auth/authorize.ts
import { auth } from '@/lib/auth';
import { WorkspaceRole, canDo, Action } from './roles';
import { NextResponse } from 'next/server';

/**
 * 权限校验装饰器
 * 
 * @param action 所需的操作权限
 * @param getResourceOwner? 可选：动态获取资源创建者ID（用于 delete_own 等操作）
 * 
 * 用法：
 *   export const GET = requireWorkspace([Action.RESOURCE_READ])(handler);
 *   export const DELETE = requireWorkspace([Action.RESOURCE_DELETE_OWN], 
 *     async (id) => { const task = await getTask(id); return task.createdBy; }
 *   )(handler);
 */
export function requireWorkspace(actions: Action[], getResourceOwner?: (params: any) => Promise<string>) {
  return function handler<T extends (...args: any[]) => Promise<any>>(fn: T) {
    return async (request: Request, context: any) => {
      // 1. 认证
      const session = await auth();
      if (!session?.user) {
        return NextResponse.json({ error: '未登录' }, { status: 401 });
      }
      
      // 2. 获取 workspaceId
      const workspaceId = extractWorkspaceId(request);
      if (!workspaceId) {
        return NextResponse.json({ error: '缺少 workspaceId' }, { status: 400 });
      }
      
      // 3. 获取角色
      const role = await getUserRole(session.user.id, workspaceId);
      if (!role) {
        return NextResponse.json({ error: '不是该工作空间的成员' }, { status: 403 });
      }
      
      // 4. 权限校验
      for (const action of actions) {
        let resourceOwnerId: string | undefined;
        if (getResourceOwner) {
          resourceOwnerId = await getResourceOwner(context.params);
        }
        
        if (!canDo(role as WorkspaceRole, action, resourceOwnerId, session.user.id)) {
          return NextResponse.json({ error: '权限不足' }, { status: 403 });
        }
      }
      
      // 5. 注入上下文并调用处理器
      return fn(request, { ...context, session, workspaceId, role });
    };
  };
}

function extractWorkspaceId(request: Request): string | null {
  // 从 header 获取
  const headerWsId = (request as any).headers?.get('x-workspace-id');
  if (headerWsId) return headerWsId;
  
  // 从 URL params 获取（Next.js App Router）
  const url = new URL(request.url);
  return url.searchParams.get('workspaceId');
}
```

```typescript
// src/lib/services/publish/content-adapter.ts

interface ArticleSource {
  title: string;
  content: string;           // HTML（insurance-d 输出格式）
  plainText: string;         // 纯文本
  images: Array<{ url: string; alt?: string }>;
  tags?: string[];
  wordCount: number;
}

/** 各平台输出格式 */
interface AdaptedContent {
  platform: string;
  title: string;
  body: string;              // 平台原生格式
  media: { urls: string[]; coverUrl?: string };
  extra: Record<string, any>; // 平台特有参数
}

class ContentAdapter {
  /**
   * 主适配入口：根据目标平台列表批量适配
   */
  adaptAll(source: ArticleSource, platforms: string[]): Record<string, AdaptedContent> {
    const result: Record<string, AdaptedContent> = {};
    
    for (const platform of platforms) {
      switch (platform) {
        case 'wechat_official':
          result[platform] = this.adaptToWechat(source);
          break;
        case 'xiaohongshu':
          result[platform] = this.adaptToXiaohongshu(source);
          break;
        case 'zhihu':
          result[platform] = this.adaptToZhihu(source);
          break;
        default:
          console.warn(`[ContentAdapter] 不支持的平台: ${platform}`);
      }
    }
    
    return result;
  }

  /** 微信公众号：HTML 格式，标题 ≤64 字符 */
  adaptToWechat(source: ArticleSource): AdaptedContent {
    return {
      platform: 'wechat_official',
      title: this.truncate(source.title, 64),
      body: source.content,  // insurance-d 输出的 HTML 直接可用
      media: {
        urls: source.images.map(img => img.url),
        coverUrl: source.images[0]?.url,
      },
      extra: {
        digest: this.extractDigest(source.plainText, 120),
        author: '',
      },
    };
  }

  /** 小红书：纯文本 + 图片，标题 ≤20 字符，正文 ≤1000 字 */
  adaptToXiaohongshu(source: ArticleSource): AdaptedContent {
    // HTML → 纯文本，去除标签
    const cleanText = this.htmlToPlainText(source.content);
    
    // 自动分段（每段不超过 200 字，小红书阅读体验最佳）
    const segments = this.segmentText(cleanText, 200);
    const body = segments.join('\n\n');
    
    return {
      platform: 'xiaohongshu',
      title: this.truncate(source.title, 20),
      body: this.truncate(body, 1000),
      media: {
        urls: source.images.slice(0, 9).map(img => img.url),
        coverUrl: source.images[0]?.url,
      },
      extra: {
        topics: (source.tags || []).slice(0, 20),
      },
    };
  }

  /** 知乎：Markdown 格式，标题 ≤100 字符 */
  adaptToZhihu(source: ArticleSource): AdaptedContent {
    return {
      platform: 'zhihu',
      title: this.truncate(source.title, 100),
      body: this.htmlToMarkdown(source.content),
      media: {
        urls: source.images.slice(0, 9).map(img => img.url),
        coverUrl: source.images[0]?.url,
      },
      extra: {
        topics: (source.tags || []).slice(0, 5),
      },
    };
  }

  // --- 工具方法 ---
  private truncate(text: string, maxLen: number): string {
    return text.length > maxLen ? text.slice(0, maxLen) : text;
  }
  
  private extractDigest(text: string, maxLen: number): string {
    // 取第一句或前 N 字作为摘要
    const firstSentence = text.split(/[。！？\n]/)[0];
    return firstSentence.length > maxLen ? firstSentence.slice(0, maxLen) + '...' : firstSentence;
  }
  
  private htmlToPlainText(html: string): string {
    return html.replace(/<[^>]+>/g, '').replace(/&nbsp;/g, ' ').replace(/\s+/g, ' ').trim();
  }
  
  private htmlToMarkdown(html: string): string {
    // 简单转换：保留 h/p/strong/em/img/list 结构
    // 生产环境建议使用 turndown 或类似库
    return html
      .replace(/<h[1-6][^>]*>(.*?)<\/h[1-6]>/gi, (_, p) => `\n## ${p}\n`)
      .replace(/<strong[^>]*>(.*?)<\/strong>/gi, '**$1**')
      .replace(/<img[^>]*src="([^"]*)"[^>]*>/gi, '![]($1)')
      .replace(/<br\s*\/?>/gi, '\n')
      .replace(/<p[^>]*>(.*?)<\/p>/gi, '$1\n\n')
      .replace(/<[^>]+>/g, '');
  }
  
  private segmentText(text: string, maxSegmentLen: number): string[] {
    const sentences = text.split(/[。！？]/);
    const segments: string[] = [];
    let current = '';
    
    for (const sentence of sentences) {
      if ((current + sentence).length > maxSegmentLen && current) {
        segments.push(current.trim());
        current = sentence;
      } else {
        current += sentence + '。';
      }
    }
    if (current) segments.push(current.trim());
    
    return segments;
  }
}
```

#### 4.4.5 发布队列服务（数据库持久化）

```typescript
// src/lib/services/publish/publish-queue.ts

/**
 * 发布队列服务
 * 
 * 设计原则：
 * - 使用数据库作为持久化存储（非内存 Map），避免重启丢失
 * - 定时 Worker 扫描 pending/scheduled 任务
 * - 失败自动重试（指数退避）
 */

class PublishQueueService {
  
  /**
   * 提交发布任务
   */
  async submit(params: {
    workspaceId: string;
    subTaskId: string;
    platforms: Array<{ platform: string; accountId: string }>;
    adaptedContents: Record<string, any>;
    scheduledAt?: Date;
    submittedBy: string;
  }): Promise<string[]> {
    const recordIds: string[] = [];
    
    for (const { platform, accountId } of params.platforms) {
      const [record] = await db.insert(publishRecords)
        .values({
          workspaceId: params.workspaceId,
          createdBy: params.submittedBy,
          subTaskId: params.subTaskId,
          platform,
          accountId,
          title: params.adaptedContents[platform]?.title || '',
          contentPreview: (params.adaptedContents[platform]?.body || '').substring(0, 500),
          adaptedContent: params.adaptedContents[platform],
          status: params.scheduledAt ? 'scheduled' : 'pending',
          scheduledAt: params.scheduledAt,
        })
        .returning();
      
      recordIds.push(record.id);
    }
    
    // 如果是立即发布，触发处理
    if (!params.scheduledAt) {
      this.processBatch(recordIds);
    }
    
    return recordIds;
  }
  
  /**
   * 处理单个发布记录
   */
  private async processRecord(recordId: string): Promise<void> {
    const [record] = await db.select()
      .from(publishRecords)
      .where(eq(publishRecords.id, recordId));
    
    if (!record || record.status !== 'pending') return;
    
    // 更新为 processing
    await db.update(publishRecords)
      .set({ status: 'publishing', updatedAt: new Date() })
      .where(eq(publishRecords.id, recordId));
    
    try {
      const provider = this.getProvider(record.platform);
      const result = await provider.publish(
        record.adaptedContent,
        record.accountId,
      );
      
      // 发布成功
      await db.update(publishRecords)
        .set({
          status: 'published',
          platformArticleId: result.articleId,
          platformUrl: result.url,
          publishedAt: new Date(),
          updatedAt: new Date(),
        })
        .where(eq(publishRecords.id, recordId));
        
      // 发送 WebSocket 通知
      await notifyPublishSuccess(record.workspaceId, record);
      
    } catch (error) {
      const retryCount = (record.retryCount || 0) + 1;
      const maxRetries = 3;
      
      if (retryCount <= maxRetries) {
        // 指数退避重试
        const delayMs = Math.pow(2, retryCount) * 5000; // 5s, 10s, 20s
        
        await db.update(publishRecords)
          .set({
            status: 'pending',  // 回到 pending，下次扫描会重新拾取
            retryCount,
            errorMessage: error.message,
            updatedAt: new Date(),
          })
          .where(eq(publishRecords.id, recordId));
          
        console.log(`[PublishQueue] 记录 ${recordId} 将在 ${delayMs}ms 后重试 (${retryCount}/${maxRetries})`);
        
      } else {
        // 超过最大重试次数，标记失败
        await db.update(publishRecords)
          .set({
            status: 'failed',
            errorMessage: error.message,
            retryCount,
            updatedAt: new Date(),
          })
          .where(eq(publishRecords.id, recordId));
          
        // 发送失败通知
        await notifyPublishFailure(record.workspaceId, record, error.message);
      }
    }
  }
  
  /**
   * 启动定时扫描 Worker
   * 
   * 由外部 cron 或 setInterval 调用
   */
  async startWorker(): Promise<void> {
    // 每 30 秒扫描一次
    setInterval(async () => {
      const now = new Date();
      
      // 1. 处理立即发布的 pending 任务（最多 10 个并发）
      const pendingRecords = await db.select()
        .from(publishRecords)
        .where(
          and(
            eq(publishRecords.status, 'pending'),
            or(
              isNull(publishRecords.scheduledAt),
              lte(publishRecords.scheduledAt, now)
            )
          )
        )
        .limit(10);
      
      for (const record of pendingRecords) {
        this.processRecord(record.id);  // 并发执行
      }
      
    }, 30 * 1000);
  }
  
  private getProvider(platform: string): PublishProvider {
    switch (platform) {
      case 'wechat_official': return new WechatPublishProvider();
      case 'xiaohongshu': return new XiaohongshuPublishProvider();
      case 'zhihu': return new ZhihuPublishProvider();
      default: throw new Error(`不支持的平台: ${platform}`);
    }
  }
}

// 单例导出
export const publishQueue = new PublishQueueService();
```

| 风险 | 等级 | 影响 | 应对措施 |
|------|------|------|----------|
| **风格模板链路断裂** | 🔴 高 | 核心功能不可用 | Phase 2 专设 2.9 验证任务；每步改造后立即测试 |

| **微信 API 调用频率限制** | 🟡 中 | 发布失败 | 队列限速 + 失败重试 + 用户告知 |
| **ContentAdapter 适配质量差** | 🟡 中 | 发布内容被平台拒 | 先做预览确认，用户手动调整后再发布 |
| **成员离职后资源归属争议** | 🟢 低 | 法律风险 | 资源归 Workspace，不随人员流动（已在设计中明确） |

## 七、验收标准

### Phase 1 验收
- [ ] 新用户注册后自动获得 Personal Workspace
- [ ] 登录后所有页面可访问，未登录跳转 `/login`
- [ ] 所有 API 返回 401（未携带有效 Session）
- [ ] WebSocket 连接需携带有效 Token
- [ ] 可切换 Workspace（如有多个）

### Phase 2 验收
- [ ] 11 张表 user_id 成功重命名为 workspace_id
- [ ] 12 张表成功添加 workspace_id 字段
- [ ] 用户 A 的任务，用户 B 完全看不到
- [ ] **风格模板链路验证**：选择账号 → 加载模板 → 生成文章，全流程正常
- [ ] `getTemplateIdForTask()` 返回正确的 templateId
- [ ] `getStyleRules()` 返回正确的规则列表

### Phase 3 验收
- [ ] Workspace Owner 可邀请/移除成员
- [ ] Editor 不能删除他人任务
- [ ] Viewer 不能创建/修改任何内容
- [ ] 前端根据角色隐藏无权按钮

### Phase 4 验收
- [ ] 任务 COMPLETED 后出现"发布"按钮
- [ ] 发布确认面板可选择平台、预览内容、设置定时

- [ ] 定时发布在指定时间后自动执行
- [ ] 发布失败自动重试 3 次
- [ ] 发布历史页面可查看所有发布记录
