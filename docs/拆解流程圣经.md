## ✅ 必须遵守的铁则

| 编号 | 铁则 | 违反后果 |
|------|------|---------|
| 1 | **任何拆解必须先存 `daily_task` 表！** | 直接存 `agent_sub_tasks` 表是严重错误！ |
| 2 | **只有一个统一流程！** | Agent B 和 insurance-d 必须走完全相同的流程！ |
| 3 | **API 入口必须统一！** | 最终只能有一个 `/api/split/confirm` API！ |
| 4 | **数据库表关系是固定的！** | `agent_tasks` → `daily_task` → `agent_sub_tasks`，顺序不能变！ |
| 5 | **任何修改前必须先看此文档！** | 不看文档就修改，出问题自己负责！ |
