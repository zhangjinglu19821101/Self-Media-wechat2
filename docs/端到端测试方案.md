# Agent 子任务执行引擎端到端测试方案

## 一、测试方案概述

### 1.1 测试目标
验证定时任务从 `agent_sub_tasks` 表提取任务到最终执行完成的完整流程，包括：
- 正常执行流程
- 异常处理与重试机制
- 最大迭代次数限制
- 交互历史记录完整性

### 1.2 测试范围

| 测试对象 | 说明 |
|---------|------|
| **4个核心功能** | 合规审核、网页搜索带摘要、网页搜索、添加草稿（微信公众号） |
| **执行引擎** | `SubtaskExecutionEngine` 完整执行链路 |
| **异常流程** | MCP失败重试、网络超时、参数错误、权限不足 |
| **边界条件** | 最大迭代次数（5次）、最大MCP尝试次数（3次） |
| **数据验证** | `agent_sub_tasks` 状态流转、`agent_sub_tasks_step_history` 记录完整性 |

### 1.3 测试环境要求

```typescript
// 测试配置
const TEST_CONFIG = {
  // 缩短等待时间用于测试
  MAX_ITERATIONS: 5,
  MAX_MCP_ATTEMPTS_PER_ITERATION: 3,
  RETRY_DELAYS: [100, 200, 400], // 毫秒，测试用短延迟
  
  // 测试数据库
  DB_CONFIG: {
    useTestDatabase: true,
    cleanupAfterTest: false // 保留数据用于验证
  }
};
```

**前置条件：**

**测试步骤：**

3. 验证执行Agent返回 `is_need_mcp=true`
4. 验证Agent B决策：type=`EXECUTE_MCP`
5. 验证MCP调用（capability_id=1，合规审核）一次成功
6. 验证Agent B决策：type=`COMPLETE`
7. 验证任务状态从 `in_progress` → `completed`

**预期结果：**
```typescript

    interact_type: 'response',
```

**验证要点：**
- [ ] 任务状态正确流转

- [ ] MCP尝试记录1次，状态success

## 四、边界条件与特殊场景测试

### 4.1 最大迭代次数测试（核心边界）

#### TC-MAX-001: 合规审核 - 恰好5次迭代后完成（边界值）

**场景设计：**
- 每次迭代执行1次MCP，前4次无法自动完成（需要更多信息）
- 第5次迭代后任务完成

**Mock配置：**
```typescript
const mockIterations = [
  { decision: 'EXECUTE_MCP', mcpResult: 'success', canComplete: false },
  { decision: 'EXECUTE_MCP', mcpResult: 'success', canComplete: false },
  { decision: 'EXECUTE_MCP', mcpResult: 'success', canComplete: false },
  { decision: 'EXECUTE_MCP', mcpResult: 'success', canComplete: false },
  { decision: 'COMPLETE', mcpResult: 'success', canComplete: true }, // 第5次完成
];
```

**预期结果：**
- 迭代次数：5次（达到但未超过最大值）
- 任务状态：`completed`
- 交互历史：10条记录（5 request + 5 response）

---

#### TC-MAX-002: 合规审核 - 超过5次迭代触发上报

**场景设计：**
- 每次迭代执行1次MCP，但始终无法自动完成
- 第5次迭代后仍无法完成，触发最大迭代限制

**Mock配置：**
```typescript
const mockIterations = Array(5).fill({
  decision: 'EXECUTE_MCP',
  mcpResult: 'success',
  canComplete: false // 始终无法自动完成
});
```

**预期结果：**
- 第5次迭代后触发 `handleMaxIterationsExceeded`
- 任务状态：`need_support`
- 交互历史最后一条：
  ```typescript
  {
    decision: { 
      type: 'FAILED', 
      reason_code: 'MAX_ITERATIONS_EXCEEDED',
      context: { execution_summary: '...' }
    },
    ext_info: { 
      max_iterations_reached: true,
      iteration: 5 
    }
  }
  ```
- 上报Agent A（验证弹框通知）

---

#### TC-MAX-003: 网页搜索 - 迭代中MCP重试不计入迭代次数

**场景设计：**
- 第1次迭代：MCP尝试3次（2次失败+1次成功）
- 第2次迭代：MCP尝试1次（成功）并完成任务

```typescript
// tests/mocks/mcp-service.mock.ts
export class MockMcpService {
  private callHistory: Map<string, any[]> = new Map();
  private mockResponses: Map<string, any> = new Map();

  // 设置Mock响应
  setMockResponse(testId: string, responses: any[]) {
    this.mockResponses.set(testId, responses);
    this.callHistory.set(testId, []);
  }

  // 模拟MCP调用
  async execute(toolName: string, action: string, params: any, testId: string): Promise<any> {
    const responses = this.mockResponses.get(testId) || [];
    const callCount = this.callHistory.get(testId)?.length || 0;
    const response = responses[callCount] || { success: false, error: { message: 'No mock response' } };
    
    this.callHistory.get(testId)?.push({ toolName, action, params, response, timestamp: Date.now() });
    
    // 模拟网络延迟
    await new Promise(resolve => setTimeout(resolve, 50));
    
    return response;
  }

  // 获取调用历史
  getCallHistory(testId: string): any[] {
    return this.callHistory.get(testId) || [];
  }

  // 清除测试数据
  clear(testId: string) {
    this.callHistory.delete(testId);
    this.mockResponses.delete(testId);
  }
}
```

### 5.3 测试工具函数

```typescript
// tests/utils/test-helpers.ts
import { db } from '@/lib/db';
import { agentSubTasks, agentSubTasksStepHistory } from '@/lib/db/schema';
import { eq } from 'drizzle-orm';

export class TestHelpers {
  // 获取任务状态
  static async getTaskStatus(taskId: string) {
    const result = await db.select().from(agentSubTasks).where(eq(agentSubTasks.id, taskId));
    return result[0];
  }

  // 获取交互历史
  static async getInteractionHistory(taskId: string) {
    return await db
      .select()
      .from(agentSubTasksStepHistory)
      .where(eq(agentSubTasksStepHistory.taskId, taskId))
      .orderBy(agentSubTasksStepHistory.stepNo, agentSubTasksStepHistory.interactNum);
  }

  // 验证交互历史完整性
  static validateInteractionHistory(history: any[], expected: {
    totalRecords: number,
    requestCount: number,
    responseCount: number,
    mcpAttempts?: number,
    hasMaxIterationsReached?: boolean
  }) {
    const requestCount = history.filter(h => h.interactType === 'request').length;
    const responseCount = history.filter(h => h.interactType === 'response').length;
    
    expect(history.length).toBe(expected.totalRecords);
    expect(requestCount).toBe(expected.requestCount);
    expect(responseCount).toBe(expected.responseCount);
    
    if (expected.mcpAttempts !== undefined) {
      const mcpAttempts = history
        .filter(h => h.interactType === 'response')
        .reduce((sum, h) => sum + (h.interactContent?.response?.mcp_attempts?.length || 0), 0);
      expect(mcpAttempts).toBe(expected.mcpAttempts);
    }
    
    if (expected.hasMaxIterationsReached) {
      const lastRecord = history[history.length - 1];
      expect(lastRecord.interactContent?.ext_info?.max_iterations_reached).toBe(true);
    }
  }

  // 模拟用户确认
  static async simulateUserConfirmation(taskId: string, confirmation: {
    keyFieldsConfirmed: any[],
    selectedSolution: any,
    userComment?: string
  }) {
    // 调用用户确认API
    await fetch(`/api/agents/user-decision`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        taskId,
        ...confirmation
      })
    });
  }

  // 清理测试数据
  static async cleanup(testId: string) {
    await db.delete(agentSubTasksStepHistory).where(eq(agentSubTasksStepHistory.taskId, testId));
    await db.delete(agentSubTasks).where(eq(agentSubTasks.id, testId));
  }
}
```
