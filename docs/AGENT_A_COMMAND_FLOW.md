# Agent A 指令下达流程（用户版）

## 🎯 标准流程

### 1. 你在 Agent A 对话框输入需求

```
用户：请再次下发全员的执行指令
```

### 2. Agent A 在对话框中整理完整的指令内容

Agent A 会生成如下格式的回复：

```markdown
## AI事业部Agent C（内容运营）
### 核心目标
1个月新增粉丝≥2000，沉淀AI公众号运营标准化能力

### 执行模块
| 执行模块 | 落地标准 | 完成时限 | 反馈要求 |
|----------|----------|----------|----------|
| 账号基建 | 3天内完成菜单栏优化 | 签收后3天 | 完成后同步测试截图 |
| 渠道获流 | 每周拓展≥3个AI社群 | 签收后30天 | 每日18:00前提交新增粉丝数 |

### 执行红线
1. 仅执行本角色专属指令，不得跨角色协同
2. 进度反馈准时率100%

---

## AI事业部Agent D（内容生成）
### 核心目标
产出优质内容支撑获流，沉淀AI内容生成全流程能力

### 执行模块
| 执行模块 | 落地标准 | 完成时限 | 反馈要求 |
|----------|----------|----------|----------|
| 前置核查 | 接收选题后2小时内核查 | 每次选题下达后2小时 | 同步核查结果 |
| 内容创作 | 每周输出4篇原创内容 | 每周五提交当周内容 | 提交内容+检测报告截图 |

---

以上指令已整理完毕，请确认是否下发
```

### 4. 你确认后系统自动发送

点击"确认下发指令"后：
- ✅ 系统自动调用指令下达 API
- ✅ 逐个发送给各个 Agent
- ✅ 通过 WebSocket 推送给各 Agent
- ✅ 保存到数据库

### 5. 其他 Agent 实时收到通知

打开其他 Agent 页面（如 Agent C），会看到：
- WebSocket 状态：在线
- 新指令通知：1 条新指令
- 指令详情：发送方（Agent A）、时间、类型、优先级、内容

## 📝 关键说明

### ✅ 你需要做的

1. 在 Agent A 对话框输入需求
2. 等待 Agent A 整理好指令内容
3. 查看确认对话框中的指令列表
4. 点击"确认下发指令"

### ✅ Agent A 会自动做的

1. 整理完整的指令内容（使用标准格式）
2. 明确列出每个 Agent 的核心目标
3. 说明执行模块、落地标准、完成时限
4. 在末尾明确说明"以上指令已整理完毕，请确认是否下发"

### ✅ 系统会自动做的

1. 检测 Agent A 的回复中是否包含指令
2. 显示确认对话框
3. 自动发送指令给各 Agent
4. 通过 WebSocket 推送通知
5. 保存到数据库

### 示例 2：向单个 Agent 下达任务

```
用户：让 Agent C 分析上周的用户数据

Agent A：
## AI事业部Agent C（内容运营）
### 核心目标
分析上周的用户活跃数据，生成分析报告

### 执行模块

|----------|----------|----------|----------|

### 执行红线
1. 数据分析必须基于真实数据
2. 报告建议必须可执行

【系统自动弹出确认对话框，用户点击确认】
✅ 指令已发送给 Agent C
```

## ⚠️ 注意事项

1. **只在 Agent A 页面启用**
   - 指令检测和确认功能仅在 Agent A 对话页面启用

2. **必须使用标准格式**
   - Agent A 的指令必须包含关键词（`##` 标题、`### 核心目标`等）
   - 必须在末尾说明"以上指令已整理完毕，请确认是否下发"

3. **需要确认**
   - 系统不会自动发送，必须由你确认
   - 可以在确认对话框中查看所有指令内容

4. **实时通知**
   - 其他 Agent 需要保持页面打开才能实时收到通知
   - 如果离线，可以在"指令历史"页面查看

## 🎉 总结

**流程非常简单：**

```
你输入需求 → Agent A 整理指令 → 你确认 → 系统自动发送
```

你只需要：
1. 告诉 Agent A 你想做什么
2. 查看 Agent A 整理的指令内容
3. 点击确认

其他的事情都交给 Agent A 和系统自动完成！🚀
