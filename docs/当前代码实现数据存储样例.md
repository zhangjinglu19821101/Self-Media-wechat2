## 样例3：最终结果（interact_type = "response"，interact_num = 2）

## 核心要点总结

| 要点 | 说明 |
|------|------|
| **成对逻辑** | request 和 response 共用同一个 `interact_num` |
| **唯一约束** | `(commandResultId, stepNo, interactNum, interactType)` |
| **MCP参数存储** | 同时存储在 `response` 和 `ext_info.mcp_call_info` 中 |
| **结构简单** | 保持当前代码的简单结构，不做过度设计 |
