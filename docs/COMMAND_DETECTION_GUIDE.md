# 指令检测与确认功能使用指南

## 📖 功能概述

在 Agent A 对话框中下达指令时，系统会自动检测指令内容，并显示确认对话框，让用户确认是否真的要发送给其他 Agent。

---

## 🎯 使用流程

### 1. 用户在 Agent A 对话框输入指令

```
用户：请再次下发全员的执行指令
```

### 2. Agent A 生成回复（流式输出）

Agent A 会生成详细的指令内容，包括：
- AI事业部Agent C（内容运营）专属指令
- AI事业部Agent D（内容生成）专属指令
- 保险事业部Agent insurance-c（内容运营）专属指令
- 保险事业部Agent insurance-d（内容创作）专属指令
- 架构师B（技术支撑）专属指令

### 3. 系统自动检测指令

前端自动检测 Agent A 的回复中是否包含指令关键词：
- `专属指令`
- `执行模块`
- `落地标准`
- `完成时限`
- `反馈要求`
- `核心目标`

### 4. 显示确认对话框

检测到指令后，显示确认对话框，内容包括：

```
┌─────────────────────────────────────────┐
│  📤 确认指令下发                        │
├─────────────────────────────────────────┤
│  检测到 Agent A 准备向以下 5 个 Agent   │
│  下达指令，请确认是否真的要发送         │
│                                          │
│  ☐ Agent C（AI运营）                   │
│     核心目标：1个月新增粉丝≥2000       │
│     状态：待发送                        │
│                                          │
│  ☐ Agent D（AI内容）                   │
│     核心目标：产出优质内容支撑获流     │
│     状态：待发送                        │
│                                          │
│  ☐ Agent insurance-c（保险运营）       │
│     核心目标：1个月新增粉丝≥1000       │
│     状态：待发送                        │
│                                          │
│  ☐ Agent insurance-d（保险内容）       │
│     核心目标：产出合规内容支撑获流     │
│     状态：待发送                        │
│                                          │
│  ☐ Agent B（技术支撑）                 │
│     核心目标：搭建技术底座              │
│     状态：待发送                        │
│                                          │
│  [取消]      [确认下发指令]            │
└─────────────────────────────────────────┘
```

### 5. 用户确认后系统自动发送

用户点击"确认下发指令"后：
1. 系统逐个调用 `POST /api/agents/send-command` API
2. 发送给对应的 Agent
3. 通过 WebSocket 推送给各 Agent
4. 保存到数据库
5. 显示发送状态（发送中/已发送/发送失败）

### 6. 其他 Agent 实时收到通知

其他 Agent 页面会显示：
- WebSocket 连接状态（在线/离线）
- 新指令通知（Toast + 黄色卡片）
- 指令详情（发送方、时间、类型、优先级、内容）

## 🔍 指令检测逻辑

### 检测关键词

系统会检测以下关键词来判断是否包含指令：
- `专属指令`
- `执行模块`
- `落地标准`
- `完成时限`
- `反馈要求`
- `核心目标`
- `指令下发`
- `全体系.*指令`
- `Agent.*指令`

### Agent 映射规则

系统会根据内容中的标题自动识别目标 Agent：

| Agent ID | 识别关键词 | 映射规则 |
|----------|-----------|---------|
| C | Agent C, 内容运营, AI事业部Agent C | 匹配标题包含 `Agent C` 或 `内容运营` |
| D | Agent D, 内容生成, AI事业部Agent D | 匹配标题包含 `Agent D` 或 `内容生成` |
| insurance-c | Agent insurance-c, 保险运营 | 匹配标题包含 `insurance-c` 或 `保险运营` |
| insurance-d | Agent insurance-d, 保险内容 | 匹配标题包含 `insurance-d` 或 `保险内容` |
| B | Agent B, 架构师, 技术支撑 | 匹配标题包含 `Agent B` 或 `架构师` |

### 指令提取规则

使用正则表达式提取每个 Agent 的指令内容：

```typescript
// 示例：提取 Agent C 的指令
/## AI事业部Agent C[\s\S]*?(?=##|$)/
```

提取后，系统会：
1. 提取核心目标
2. 判断优先级（高/普通/低）
3. 设置指令类型（默认为 task）

## 📤 指令发送流程

### 1. 格式化指令内容

系统会将 Agent A 的回复格式化为标准指令格式：

```text
【收到来自 Agent A 的指令】

指令类型：task
优先级：high
发送时间：2026/2/2 20:00:00

指令内容：
核心目标：1个月新增粉丝≥2000
执行模块：账号基建、渠道获流、用户运营、能力沉淀
...

---
请按照指令要求执行任务，执行完成后向 Agent A 报告结果。
```

### 2. 调用 API 发送

```bash
POST /api/agents/send-command
Content-Type: application/json

{
  "fromAgentId": "A",
  "toAgentId": "C",
  "command": "【收到来自 Agent A 的指令】...",
  "commandType": "task",
  "priority": "high"
}
```

### 3. WebSocket 推送

API 成功后，会通过 WebSocket 推送给目标 Agent：

```json
{
  "type": "new_command",
  "fromAgentId": "A",
  "toAgentId": "C",
  "command": "【收到来自 Agent A 的指令】...",
  "commandType": "task",
  "priority": "high",
  "timestamp": "2026-02-02T12:00:00.000Z",
  "data": {
    "conversationId": "xxx",
    "sessionId": "agent-A-to-C-xxx"
  }
}
```

### 4. 保存到数据库

指令会保存到数据库的 `conversations` 和 `messages` 表，支持：
- 指令历史查询
- 状态跟踪
- 结果反馈

---

## ✨ 特性

1. **自动检测**：无需手动触发，系统自动识别指令
2. **用户确认**：显示确认对话框，避免误发
3. **实时推送**：通过 WebSocket 实时通知
4. **状态跟踪**：显示发送状态（发送中/已发送/失败）
5. **错误处理**：发送失败时显示错误信息
6. **批量发送**：支持一次性发送给多个 Agent

## 🛠️ 技术实现

### 前端组件

1. **指令检测器** (`src/lib/command-detector.ts`)
   - 检测 Agent A 回复中的指令
   - 提取目标 Agent 和指令内容
   - 格式化指令内容

2. **确认对话框** (`src/components/command-confirm-dialog.tsx`)
   - 显示待发送的指令列表
   - 逐个发送指令
   - 显示发送状态

3. **Agent 对话页面** (`src/app/agents/[id]/page.tsx`)
   - 集成指令检测功能
   - 显示确认对话框

### 后端 API

1. **指令下达 API** (`src/app/api/agents/send-command/route.ts`)
   - 接收指令请求
   - 保存到数据库
   - WebSocket 推送
   - 返回流式响应

### WebSocket 服务

1. **WebSocket 服务器** (`src/lib/websocket-server.ts`)
   - 监听端口 5001
   - 管理客户端连接
   - 推送指令通知

## 📝 注意事项

1. **仅在 Agent A 页面启用**
   - 指令检测和确认功能仅在 Agent A 对话页面启用
   - 其他 Agent 页面不会显示确认对话框

2. **需要 WebSocket 连接**
   - 其他 Agent 需要保持 WebSocket 连接才能实时收到通知
   - 如果离线，可以到"指令历史"页面查看

3. **指令格式规范**
   - Agent A 的回复需要按照标准格式（包含关键词）
   - 标题需要清晰标注目标 Agent

4. **网络要求**
   - 需要稳定的网络连接
   - 发送失败时会显示错误信息

## 🚀 快速测试

### 测试步骤

1. 打开 Agent A 对话页面：`http://localhost:5000/agents/A`

2. 输入测试指令：
   ```
   请再次下发全员的执行指令
   ```

3. 等待 Agent A 生成回复

4. 确认对话框自动弹出

5. 点击"确认下发指令"

6. 查看发送状态

7. 打开其他 Agent 页面（如 Agent C），查看是否收到通知

### 验证方法

- Agent A 页面：确认对话框显示正确
- Agent C/D/insurance-c/insurance-d/B 页面：显示新指令通知
- 指令历史页面：可以查看已发送的指令
- 数据库：可以查询指令记录
