# 多微信公众号管理指南

## 📋 概述

本项目支持**多个微信公众号**的配置和管理，通过智能匹配机制自动选择合适的公众号进行操作。

## 🏗️ 架构设计

### 1. 配置文件位置
```
src/config/wechat-official-account.config.ts
```

### 2. 数据结构

每个公众号配置包含以下字段：

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `id` | string | 账号唯一标识（用于程序调用） | `insurance-account` |
| `name` | string | 公众号名称（用于展示） | `保险科普公众号` |
| `appId` | string | 微信公众号 AppID | `wxdb3ea2f8e0bb2496` |
| `appSecret` | string | 微信公众号 AppSecret（敏感信息） | `9dffee725f3ce4efbd66991a01898727` |
| `agent` | string | 绑定的 Agent 类型 | `insurance-d` \| `agent-d` \| `both` |
| `description` | string | 账号描述 | `insurance-d 对应的保险科普公众号` |
| `enabled` | boolean | 是否启用 | `true` |
| `defaultAuthor` | string | 默认作者 | `保险科普` |
| `defaultAuthorId` | number | 默认作者ID | `1` |

## 🔄 账号选择机制

### 选择优先级（从高到低）

```
1. 明确指定 accountId 
   ↓
2. 从任务内容中提取 accountId
   ↓
3. 根据 Agent 类型自动选择默认账号
   ↓
4. 使用第一个可用账号（兜底）
```

### 详细说明

#### 1️⃣ 明确指定 accountId（最高优先级）

**使用场景**：调用方明确知道要使用哪个账号

**示例代码**：
```typescript
import { getAccountById } from '@/config/wechat-official-account.config';
import { wechatAddDraft } from '@/lib/mcp/wechat-tools';

// 方式1：直接通过 ID 获取账号
const account = getAccountById('insurance-account');
if (account) {
  console.log('使用账号:', account.name);
  console.log('AppID:', account.appId);
  // appSecret 会被自动使用，不需要手动处理
}

// 方式2：在 MCP 调用时指定 accountId
const result = await wechatAddDraft({
  accountId: 'insurance-account',  // 明确指定
  articles: [...],
});
```

#### 2️⃣ 从任务内容中提取 accountId

**使用场景**：任务描述中包含账号信息

**支持的格式**：
- `公众号账号ID：insurance-account`
- `accountId: ai-tech-account`
- `使用账号：insurance-account`
- `上传到：ai-tech-account`

**示例代码**：
```typescript
import { getTaskAccountId } from '@/lib/utils/account-id-extractor';

// 从任务内容中提取
const taskContent = '无法上传微信公众号文章草稿，公众号账号ID：insurance-account';
const result = getTaskAccountId(taskContent, 'insurance-d');

if (result.success) {
  console.log('提取到的账号ID:', result.accountId);
  console.log('获取方式:', result.method); // 'extracted'
}
```

#### 3️⃣ 根据 Agent 类型自动选择（最常用）

**使用场景**：不需要明确指定，根据 Agent 类型自动选择

**映射关系**：
| Agent 类型 | 默认账号ID | 说明 |
|-----------|-----------|------|
| `insurance-d` | `insurance-account` | 保险事业部公众号 |
| `agent-d` | `ai-tech-account` | AI 技术公众号 |

**示例代码**：
```typescript
import { getAgentAccount, matchAccount } from '@/config/wechat-official-account.config';
import { getTaskAccountId } from '@/lib/utils/account-id-extractor';

// 方式1：直接获取 Agent 的默认账号
const account = getAgentAccount('insurance-d');
if (account) {
  console.log('insurance-d 的默认账号:', account.name);
}

// 方式2：使用智能匹配
const matched = matchAccount({ agent: 'agent-d' });
if (matched) {
  console.log('匹配到的账号:', matched.name);
}

// 方式3：使用便捷函数
const result = getTaskAccountId(undefined, 'insurance-d');
if (result.success) {
  console.log('自动选择的账号:', result.accountId);
  console.log('获取方式:', result.method); // 'agent_default'
}
```

#### 4️⃣ 使用第一个可用账号（兜底）

**使用场景**：以上方式都失败时的最后备选

**示例代码**：
```typescript
import { getEnabledAccounts, listAvailableAccounts } from '@/config/wechat-official-account.config';

// 获取所有启用的账号
const allAccounts = getEnabledAccounts();
console.log('所有可用账号:', allAccounts.map(a => a.name));

// 获取账号选项列表（用于UI展示）
const options = listAvailableAccounts();
console.log('账号选项:', options);
/*
[
  {
    value: 'insurance-account',
    label: '保险科普公众号 (insurance-account)',
    description: 'insurance-d 对应的保险科普公众号',
    agent: 'insurance-d'
  },
  {
    value: 'ai-tech-account',
    label: 'AI技术公众号 (ai-tech-account)',
    description: 'Agent D 对应的 AI 技术公众号',
    agent: 'agent-d'
  }
]
*/
```

## 📝 添加新公众号

### 步骤1：在配置文件中添加

编辑 `src/config/wechat-official-account.config.ts`：

```typescript
export const defaultWechatConfig: Record<string, WechatOfficialAccount> = {
  // 现有的账号...
  'insurance-account': { ... },
  'ai-tech-account': { ... },
  
  // 新增的公众号
  'new-business-account': {
    id: 'new-business-account',
    name: '新业务公众号',
    appId: 'wx1234567890abcdef',      // 从微信公众平台获取
    appSecret: 'your-app-secret-here',    // 从微信公众平台获取
    agent: 'insurance-d',                   // 或 'agent-d' 或 'both'
    description: '新业务线的公众号',
    enabled: true,
    defaultAuthor: '新业务',
    defaultAuthorId: 3,
    createdAt: Date.now(),
    updatedAt: Date.now(),
  },
};
```

### 步骤2：配置环境变量（可选）

如果不想把 AppSecret 硬编码在代码中，可以使用环境变量：

编辑 `.env.local`：
```env
# 新业务公众号配置
WECHAT_NEW_BUSINESS_APP_ID=wx1234567890abcdef
WECHAT_NEW_BUSINESS_APP_SECRET=your-app-secret-here
```

然后修改配置文件读取环境变量。

## 🔐 安全注意事项

### 1. AppSecret 保密

❌ **错误做法**：
```typescript
// 不要在日志中打印 appSecret
console.log('账号信息:', account); // 会暴露 appSecret！
```

✅ **正确做法**：
```typescript
// 使用 wechat-tools.ts 中的函数，它会自动隐藏敏感信息
const result = await wechatGetAccounts();
// 返回的数据中 appSecret 会被替换为 '***'

// 或者手动处理
const safeAccount = {
  ...account,
  appSecret: '***',
};
```

### 2. 环境变量存储敏感信息

推荐将 AppID 和 AppSecret 存储在环境变量中，而不是代码中。

## 💡 最佳实践

### 1. 默认使用 Agent 自动匹配

大多数情况下，不需要明确指定账号，让系统根据 Agent 类型自动选择：

```typescript
// ✅ 推荐：简洁、不易出错
const result = getTaskAccountId(undefined, 'insurance-d');

// ❌ 不推荐：除非确实需要指定特定账号
const result = getTaskAccountId(undefined, 'insurance-d', 'insurance-account');
```

### 2. 任务内容中明确账号

如果有多个账号可用，建议在任务内容中明确说明：

```typescript
// ✅ 好的做法
const taskContent = '上传文章到公众号账号ID：insurance-account';

// ❌ 不好的做法
const taskContent = '上传文章'; // 系统需要猜测
```

### 3. 定期检查账号配置

```typescript
import { getEnabledAccounts } from '@/config/wechat-official-account.config';

// 检查所有启用的账号
const accounts = getEnabledAccounts();
console.log('当前启用的公众号数量:', accounts.length);
accounts.forEach(acc => {
  console.log(`- ${acc.name} (${acc.id}): ${acc.enabled ? '✅' : '❌'}`);
});
```
