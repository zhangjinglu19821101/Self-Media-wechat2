# 更灵活的 MCP 工具注册方案

## 😅 问题

如果每次新增工具都要改 `layout.tsx`，那和方式 1 没什么区别！

---

## 🎯 方案 1：插件自动加载（推荐）

### 核心思路

创建一个插件目录，里面的文件自动注册，不需要改任何代码！

### 步骤 1：创建插件目录结构

```
src/lib/mcp/
├── plugins/
│   ├── index.ts          # 插件加载器
│   ├── email-plugin.ts   # 邮件工具插件
│   ├── weather-plugin.ts # 天气工具插件
│   └── ...
```

### 步骤 2：创建插件加载器

```typescript
// src/lib/mcp/plugins/index.ts

import { toolRegistry } from '../tool-registry';
import * as fs from 'fs';
import * as path from 'path';

/**
 * MCP 插件接口
 */
export interface MCPPlugin {
  name: string;
  description?: string;
  tools: any;
}

/**
 * 自动加载 plugins 目录下的所有插件
 */
export async function loadMCPPlugins() {
  console.log('[Plugin Loader] 开始加载 MCP 插件...');
  
  const pluginsDir = path.join(__dirname);
  const files = fs.readdirSync(pluginsDir);
  
  for (const file of files) {
    // 跳过 index.ts 和非 .ts 文件
    if (file === 'index.ts' || !file.endsWith('-plugin.ts')) {
      continue;
    }
    
    try {
      const pluginPath = path.join(pluginsDir, file);
      const pluginModule = await import(pluginPath);
      
      if (pluginModule.default && pluginModule.default.name) {
        const plugin: MCPPlugin = pluginModule.default;
        
        toolRegistry.registerTool(
          plugin.name,
          plugin.tools,
          plugin.description
        );
        
        console.log(`[Plugin Loader] ✅ 插件加载成功: ${plugin.name}`);
      }
    } catch (error) {
      console.error(`[Plugin Loader] ❌ 插件加载失败: ${file}`, error);
    }
  }
  
  console.log('[Plugin Loader] 插件加载完成');
  console.log('[Plugin Loader] 可用工具:', toolRegistry.getAvailableTools());
}
```

### 步骤 3：创建插件示例

```typescript
// src/lib/mcp/plugins/email-plugin.ts

export interface SendEmailParams {
  to: string;
  subject: string;
  body: string;
}

const EmailTools = {
  async sendEmail(params: SendEmailParams) {
    console.log('[Email Plugin] 发送邮件:', params);
    return { success: true };
  }
};

export default {
  name: 'email',
  description: '邮件相关工具：发送邮件',
  tools: EmailTools,
};
```

### 步骤 4：在 layout.tsx 中调用

```typescript
// src/app/layout.tsx

import { loadMCPPlugins } from '@/lib/mcp/plugins';

// 启动后台服务（仅在服务端运行）
if (typeof window === 'undefined') {
  // ... 其他初始化代码 ...
  
  // ============================================
  // 🔥 自动加载所有 MCP 插件！
  // ============================================
  loadMCPPlugins();
}
```

### 使用方式

**新增工具只需要：**

1. ✅ 在 `src/lib/mcp/plugins/` 目录下创建新文件 `xxx-plugin.ts`
2. ✅ 重启应用（因为是应用启动时加载）
3. ✅ **不需要改任何其他代码！**

## 🎯 方案 2：API 动态注册（最灵活，不需要重启）

### 核心思路

创建一个 API 路由，可以在运行时动态注册工具，**完全不需要重启！**

### 步骤 1：创建 API 路由

```typescript
// src/app/api/mcp/register-tool/route.ts

import { toolRegistry } from '@/lib/mcp/tool-registry';
import { NextRequest, NextResponse } from 'next/server';

export async function POST(request: NextRequest) {
  try {
    const { name, tools, description } = await request.json();
    
    if (!name || !tools) {
      return NextResponse.json(
        { success: false, error: '缺少必需参数：name 和 tools' },
        { status: 400 }
      );
    }
    
    toolRegistry.registerTool(name, tools, description);
    
    console.log(`[API] ✅ 工具动态注册成功: ${name}`);
    console.log('[API] 当前可用工具:', toolRegistry.getAvailableTools());
    
    return NextResponse.json({
      success: true,
      message: `工具 ${name} 注册成功`,
      availableTools: toolRegistry.getAvailableTools(),
    });
  } catch (error: any) {
    console.error('[API] ❌ 工具注册失败:', error);
    return NextResponse.json(
      { success: false, error: error.message },
      { status: 500 }
    );
  }
}

export async function GET() {
  return NextResponse.json({
    success: true,
    availableTools: toolRegistry.getAvailableTools(),
  });
}
```

### 步骤 2：使用方式

**动态注册工具（不需要重启！）：**

```bash
# 调用 API 注册工具
curl -X POST http://localhost:5000/api/mcp/register-tool \
  -H "Content-Type: application/json" \
  -d '{
    "name": "email",
    "description": "邮件工具",
    "tools": {
      "sendEmail": function(params) {
        console.log("发送邮件:", params);
        return { success: true };
      }
    }
  }'
```

**查看已注册的工具：**

```bash
curl http://localhost:5000/api/mcp/register-tool
```

## 🎯 方案 3：配置文件驱动（平衡灵活性和简单性）

### 核心思路

用 JSON 配置文件定义工具，启动时自动加载。

### 步骤 1：创建配置文件

```json
// config/mcp-tools.json
{
  "tools": [
    {
      "name": "email",
      "description": "邮件相关工具",
      "module": "@/lib/mcp/email-tools"
    },
    {
      "name": "weather",
      "description": "天气相关工具",
      "module": "@/lib/mcp/weather-tools"
    }
  ]
}
```

### 步骤 2：创建配置加载器

```typescript
// src/lib/mcp/config-loader.ts

import { toolRegistry } from './tool-registry';
import * as fs from 'fs';
import * as path from 'path';

interface ToolConfig {
  name: string;
  description?: string;
  module: string;
}

interface MCPToolsConfig {
  tools: ToolConfig[];
}

export async function loadToolsFromConfig() {
  console.log('[Config Loader] 从配置文件加载 MCP 工具...');
  
  const configPath = path.join(process.cwd(), 'config', 'mcp-tools.json');
  
  if (!fs.existsSync(configPath)) {
    console.log('[Config Loader] 配置文件不存在，跳过');
    return;
  }
  
  const configContent = fs.readFileSync(configPath, 'utf-8');
  const config: MCPToolsConfig = JSON.parse(configContent);
  
  for (const toolConfig of config.tools) {
    try {
      const toolModule = await import(toolConfig.module);
      
      toolRegistry.registerTool(
        toolConfig.name,
        toolModule.default || toolModule,
        toolConfig.description
      );
      
      console.log(`[Config Loader] ✅ 工具加载成功: ${toolConfig.name}`);
    } catch (error) {
      console.error(`[Config Loader] ❌ 工具加载失败: ${toolConfig.name}`, error);
    }
  }
  
  console.log('[Config Loader] 配置加载完成');
}
```

## 📊 三种方案对比

| 维度 | 方案 1：插件自动加载 | 方案 2：API 动态注册 | 方案 3：配置文件驱动 |
|------|------------------|------------------|------------------|
| **新增工具需要改代码吗？** | ❌ 不需要（新建文件） | ❌ 不需要（调用 API） | ❌ 不需要（改配置） |
| **需要重启吗？** | ✅ 需要 | ❌ 不需要 | ✅ 需要 |
| **灵活性** | ⭐⭐⭐⭐ 高 | ⭐⭐⭐⭐⭐ 最高 | ⭐⭐⭐ 中 |
| **复杂度** | ⭐⭐ 中 | ⭐⭐⭐⭐ 高 | ⭐ 低 |
| **推荐度** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐ |

## 🎯 我的推荐

| 场景 | 推荐方案 |
|------|---------|
| **团队协作，需要规范** | 方案 1（插件自动加载） |
| **需要极高灵活性，经常变更** | 方案 2（API 动态注册） |
| **简单场景，工具不多** | 方案 3（配置文件驱动） |
| **最平衡的选择** | **方案 1（插件自动加载）** |

## ✅ 总结

**核心原则：**

1. ✅ **不要每次都改 `layout.tsx`** - 那样和方式 1 没区别
2. ✅ **选择一个更灵活的方案** - 插件自动加载、API 动态注册、配置文件驱动
3. ✅ **方案 1（插件自动加载）是最平衡的选择** - 推荐使用！
