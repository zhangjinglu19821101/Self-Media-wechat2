# 关键调用点分析文档

## 概述
## 需要添加 `recordAgentInteraction()` 和 `recordMcpExecution()` 调用的关键位置

## 一、关键位置分析

### 1. **执行 Agent 调用位置

**位置**: `sendBackToExecutor()` 方法 (src/lib/services/subtask-execution-engine.ts:2804`

**场景**: 调用执行 Agent（insurance-d 等）后

**状态**:
- **正常情况**: `pre_completed` 或 `pre_need_support`
- **异常情况**: `pre_failed`

**代码**:
```typescript
public async sendBackToExecutor(...) {
  try {
    const response = await callLLM(...);
    
    // 🔴 这里需要添加 recordAgentInteraction() 调用 - 正常情况
    // 状态: pre_completed 或 pre_need_support
    
    return { finalResult: response };
  } catch (error) {
    console.error('[SubtaskEngine] 返回给执行Agent失败:', error);
    
    // 🔴 这里需要添加 recordAgentInteraction() 调用 - 异常情况
    // 状态: pre_failed
    
    return { finalResult: '执行完成', mcpResult };
  }
}
```

### 3. **Agent B 决策位置**

**位置**: `callAgentBWithDecision() 方法 (src/lib/services/subtask-execution-engine.ts:3237`

**场景**: Agent B 返回后

**状态**:
- `EXECUTE_MCP`

- `NEED_USER`
- `FAILED`
- `REEXECUTE_EXECUTOR`

**代码**:

public async callAgentBWithDecision(...) {
  try {

    // 🔴 这里需要添加 recordAgentInteraction() 调用

  } catch (error) {
    // 🔴 这里也需要处理异常
  }

## 二、详细实施计划

### Phase 1: sendBackToExecutor() 方法

1. **正常情况**
- 在 `const response = await callLLM(...)` 之后添加
- 状态: 根据 executorResult.isTaskDown ? 'pre_completed' : 'pre_need_support'

2. **异常情况**
- 在 catch 块中添加
- 状态: 'pre_failed'

### Phase 2: executeCapability() 方法

1. **正常情况**
- 在 `const mcpResult = await genericMCPCall(...)` 之后添加

2. **异常情况**
- 在 catch 块中添加

### Phase 3: callAgentBWithDecision() 方法

1. **正常情况**
- 在 `const response = await callLLM(...)` 之后添加
- 状态: 根据 agentBDecision.type 确定
