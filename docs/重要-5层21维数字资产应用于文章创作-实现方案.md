# 5层21维数字资产应用于文章创作 — 实现方案

> 评审版本 v1.0 | 2025-07-14

---

## 一、现状分析

### 1.1 什么是「5层21维」

通过 `article-extraction-service.ts` 对用户上传的文章进行 AI 提取，产生5个层级的结构化数字资产：

| 层级 | 名称 | 存储表 | 维度数 | 说明 |
|------|------|--------|--------|------|
| L1 | 文章元信息 | `article_extractions` | 4维 | 文章类型/行业标识/风格基调/平台 |
| L2 | 结构资产 | `extraction_assets` (layer=structure) | 5维 | 开头模式/段落节奏/转折模式/收尾模式/结构模板 |
| L3 | 修辞资产 | `extraction_assets` (layer=rhetoric) | 4维 | 类比用法/数据引用/反问句式/金句模式 |
| L4 | 认知资产 | `extraction_assets` (layer=cognitive) | 5维 | 误区类型/破局路径/认知框架/读者画像/情绪节奏 |
| L5 | 行业资产 | `extraction_assets` (layer=industry) | 3维 | 险种关联/条款引用/行业术语 |

**合计：4+5+4+5+3 = 21维**

### 1.2 当前数据流（断裂状态）

```
用户上传文章
  → article-extraction-service 提取 5层21维
    → 存入 article_extractions + extraction_assets 表
      → 用户点"保存到素材库"
        → 部分维度转入 material_library 表（case/analogy/law等类型）
          → ❌ 断裂点：提取了21维，但只有3-5维被真正用于创作提示词
```

### 1.3 当前提示词注入清单

| 注入项 | 来源 | 对应维度 | 覆盖层级 |
|--------|------|---------|---------|
| 风格规则 (styleRules) | style_assets 表 | 用词/句式/语气 | L2部分 |
| 行业案例 (industryCases) | industry_case_library 表 | 理赔案例 | L5部分 |
| 类比素材 (analogyMaterials) | material_library (sceneType) | 类比案例 | L3部分 |
| 用户观点 (userOpinion) | 前端创作引导 | 核心观点 | 无 |
| 素材 (materials) | material_library | 案例素材 | L5部分 |
| 创作类型 (articleType) | metadata | 文章类型 | L1部分 |
| 结构模板 (articleStructureTemplate) | article-structure-templates | 结构模板 | L2部分 |

**覆盖率：21维中仅约7维被使用（33%），14维处于沉睡状态。**

### 1.4 未被使用的14维

| 维度 | 所属层级 | 当前状态 | 潜在价值 |
|------|---------|---------|---------|
| 行业标识 | L1 | ❌ 未注入 | 可自动匹配行业术语和案例 |

## 二、核心设计：按创作节点注入维度

### 2.1 创作流程节点与维度映射

### 2.2 各节点注入策略详解

#### Step 1：分析节点（Agent B / 通用分析）

**目标**：让分析 Agent 理解文章的行业背景、目标读者、认知框架

| 注入维度 | 来源 | 注入方式 | 作用 |
|---------|------|---------|------|
| 行业标识 (L1) | extraction_assets | 注入到分析提示词 | 识别文章涉及的险种领域 |
| 风格基调 (L1) | extraction_assets | 注入到分析提示词 | 判断文章情感定位 |
| 读者画像 (L4) | extraction_assets | 注入到分析提示词 | 分析目标受众特征 |
| 认知框架 (L4) | extraction_assets | 注入到分析提示词 | 选择说服策略 |

**数据流**：
```
buildExecutionContext()
  → 查询 extraction_assets (workspaceId + layer IN ['meta', 'cognitive'])
    → 格式化为 analysisContext 文本
      → 注入到 Agent B 的 priorStepOutput 或 userPrompt
```

#### Step 2：大纲节点（insurance-d 生成大纲）

**目标**：让大纲遵循用户的历史文章结构模式

| 注入维度 | 来源 | 注入方式 | 作用 |
|---------|------|---------|------|
| 开头模式 (L2) | extraction_assets | 注入到大纲提示词 | 指导大纲的开头设计 |
| 段落节奏 (L2) | extraction_assets | 注入到大纲提示词 | 控制大纲的段落结构 |
| 转折模式 (L2) | extraction_assets | 注入到大纲提示词 | 安排大纲的转折节点 |
| 收尾模式 (L2) | extraction_assets | 注入到大纲提示词 | 指导大纲的结尾设计 |
| 误区类型 (L4) | extraction_assets | 注入到大纲提示词 | 如果是误区型，规划破局路径 |

**数据流**：
```
assemblePrompt() → formatCurrentTask()
  → 当 subTaskRole === 'outline_generation' 时
    → 查询 extraction_assets (workspaceId + layer = 'structure')
      → 格式化为 structureGuidance 文本
        → 追加到 currentTask 区域
```

#### Step 3：写作节点（insurance-d 生成全文）— **核心注入点**

**目标**：全量注入5层21维，让AI完全复刻用户的写作风格

| 注入维度 | 来源 | 注入方式 | 作用 |
|---------|------|---------|------|
| **L1 行业标识** | extraction_assets | 拼接到创作类型声明 | 自动匹配行业术语表 |

| **L1 平台** | extraction_assets | 拼接到平台适配区 | 精准适配平台风格 |
| **L2 开头模式** | extraction_assets | 拼接到结构模板 | 复刻开头写法 |
| **L2 段落节奏** | extraction_assets | 拼接到篇幅要求 | 控制长短段交替 |
| **L2 转折模式** | extraction_assets | 拼接到结构模板 | 复刻转折写法 |
| **L2 收尾模式** | extraction_assets | 拼接到结构模板 | 复刻结尾写法 |
| **L2 结构模板** | extraction_assets | 拼接到结构模板 | 复刻整体结构 |
| **L3 类比用法** | extraction_assets | 拼接到类比素材区 | 指导类比写法 |
| **L3 数据引用** | extraction_assets | 拼接到素材区 | 指导数据嵌入方式 |
| **L3 反问句式** | extraction_assets | 拼接到修辞指导区 | 复刻反问句风格 |
| **L3 金句模式** | extraction_assets | 拼接到修辞指导区 | 生成风格一致的金句 |
| **L4 误区类型** | extraction_assets | 拼接到创作类型区 | 匹配破局路径 |
| **L4 破局路径** | extraction_assets | 拼接到创作类型区 | 指导反驳策略 |
| **L4 认知框架** | extraction_assets | 拼接到认知指导区 | 选择说服策略 |
| **L4 读者画像** | extraction_assets | 拼接到读者定位区 | 调整语气深度 |
| **L4 情绪节奏** | extraction_assets | 拼接到情绪指导区 | 控制情绪曲线 |
| **L5 险种关联** | extraction_assets | 拼接到行业知识区 | 自动匹配险种知识 |
| **L5 条款引用** | extraction_assets | 拼接到法规区 | 自动关联法规条款 |
| **L5 行业术语** | extraction_assets | 拼接到术语区 | 控制专业术语密度 |

#### Step 4：预览修改节点（user_preview_edit）

**目标**：辅助用户判断文章是否符合自己的风格

| 注入维度 | 来源 | 注入方式 | 作用 |
|---------|------|---------|------|

| 情绪节奏 (L4) | extraction_assets | 前端展示对比 | 提示用户情绪曲线是否一致 |

**目标**：用行业资产辅助合规校验

| 注入维度 | 来源 | 注入方式 | 作用 |

#### Step 6：上传节点（无额外注入）

### 3.2 新增服务：ExtractionAssetInjector

```typescript
// src/lib/services/extraction-asset-injector.ts

interface AssetInjectionOptions {
  workspaceId: string;
  articleType?: string;          // 创作类型（筛选匹配的资产）
  layers: AssetLayer[];          // 需要查询的层级
  executorType?: string;         // 执行Agent类型（影响平台适配）
  maxAssetsPerLayer?: number;    // 每层最多返回几条（默认3）
}

type AssetLayer = 'meta' | 'structure' | 'rhetoric' | 'cognitive' | 'industry';

interface InjectionResult {
  // 按注入区域分组的格式化文本
  industryContext: string;       // 行业上下文（L1+L5）
  structureGuidance: string;     // 结构指导（L2）
  rhetoricGuidance: string;      // 修辞指导（L3）
  cognitiveGuidance: string;     // 认知指导（L4）
  emotionGuidance: string;       // 情绪指导（L4情绪子维度）
  
  // 元信息（供调试和日志）
  assetCount: number;
  layerCoverage: string[];       // ['L1', 'L2', 'L3', 'L4', 'L5']
  fromCache: boolean;
}

class ExtractionAssetInjector {
  private cache: Map<string, { result: InjectionResult; expireAt: number }> = new Map();
  private CACHE_TTL = 5 * 60 * 1000; // 5分钟

  /**
   * 查询并格式化数字资产
   */
  async inject(options: AssetInjectionOptions): Promise<InjectionResult>;
  
  /**
   * 按层查询资产（内部方法）
   */
  private async queryAssetsByLayer(
    workspaceId: string, 
    layer: AssetLayer, 
    articleType?: string,
    limit?: number
  ): Promise<ExtractionAsset[]>;
  
  /**
   * 格式化 L1 元信息层
   * 输出示例：
   * 【行业标识】保险-重疾险-理赔
   * 【风格基调】共情式破局，先接纳再点破
   * 【目标平台】微信公众号
   */
  private formatMetaLayer(assets: ExtractionAsset[]): string;
  
  /**
   * 格式化 L2 结构层
   * 输出示例：
   * 【开头模式】先用客户原话引入，再承认逻辑合理
   * 【段落节奏】长短交替，短段不超过80字，长段200-400字
   * 【转折模式】"但我想请你多想一步：" / "问题来了——"
   * 【收尾模式】灵魂拷问 + 价值重构
   */
  private formatStructureLayer(assets: ExtractionAsset[]): string;
  
  /**
   * 格式化 L3 修辞层
   * 输出示例：
   * 【类比用法】先描述日常场景，再映射保险原理。例：闹钟→保障价值
   * 【数据引用】引用行业数据时标注具体年份和机构
   * 【反问句式】"你真会觉得XX是赚到吗？"
   * 【金句模式】短句+对比结构。例："保险不是投资工具，至少保障类的不是"
   */
  private formatRhetoricLayer(assets: ExtractionAsset[]): string;
  
  /**
   * 格式化 L4 认知层
   * 输出示例：
   * 【误区类型】回本逻辑：用投资标准评价保障
   * 【破局路径】先承认→再标准错位→然后本质澄清→最后类比讲透
   * 【认知框架】标准错位型（A标准评价B事物）
   * 【读者画像】30-45岁中产，有保险意识但认知不深
   */
  private formatCognitiveLayer(assets: ExtractionAsset[]): string;
  
  /**
   * 格式化 L5 行业层
   * 输出示例：
   * 【险种关联】重疾险、医疗险
   * 【条款引用】《保险法》第十六条 如实告知义务
   * 【行业术语】使用通俗解释+括号备注专业术语
   */
  private formatIndustryLayer(assets: ExtractionAsset[]): string;
}
```

## 四、查询策略设计

### 4.1 资产查询优先级

```
1. 精确匹配：workspaceId + articleType + layer + dimension
   ↓ 无结果
2. 类型匹配：workspaceId + articleType + layer（不限 dimension）
   ↓ 无结果
3. 工作区匹配：workspaceId + layer（不限 articleType）
   ↓ 无结果
4. 系统默认：ownerType='system' + layer
   ↓ 无结果
5. 跳过注入（不污染提示词）
```

### 4.2 资产去重与优先级

- 同一维度（如同比用法）可能有多条资产（来自不同文章的提取）
- 策略：取最新的一条（按 createdAt DESC），除非有 usageCount 更高的
- 格式化时只输出最精炼的1条，避免提示词膨胀

### 4.3 提示词长度控制

| 注入区域 | 最大长度 | 超出策略 |
|---------|---------|---------|
| 行业上下文 (L1+L5) | 300字 | 截断，保留核心标签 |
| 结构指导 (L2) | 500字 | 每维度最多1条，截断描述 |
| 修辞指导 (L3) | 400字 | 每维度最多1条示例 |
| 认知指导 (L4) | 400字 | 每维度最多1条 |
| 情绪指导 (L4) | 200字 | 只保留节奏描述 |
| **合计** | **≤1800字** | 约占提示词总量5-8% |

## 八、风险评估

| 风险 | 等级 | 对策 |
|------|------|------|
| 提示词过长导致AI输出质量下降 | 🟠 中 | 严格控制注入文本≤1800字，每维度最多1条 |
| extraction_assets 数据不足 | 🟡 低 | 4级降级策略（精确→类型→工作区→系统→跳过） |
| 缓存导致资产更新不及时 | 🟡 低 | 5分钟TTL，素材库更新时主动清缓存 |
| 与现有风格规则(styleRules)冲突 | 🟠 中 | 数字资产描述"风格倾向"，风格规则描述"硬性约束"，后者优先 |
| 同一维度多条资产冲突 | 🟡 低 | 取最新1条，格式化时不叠加 |

## 九、与现有系统的兼容性

| 现有功能 | 关系 | 说明 |
|---------|------|------|
| 风格规则 (styleRules) | 互补 | 风格规则=硬约束，数字资产=风格倾向，不冲突 |
| 行业案例 (industryCases) | 互补 | 行业案例=具体案例，数字资产L5=行业知识框架 |
| 类比素材 (analogyMaterials) | 互补 | 类比素材=具体类比，数字资产L3=类比写法指导 |
| 创作类型 (articleType) | 增强 | 创作类型=结构骨架，数字资产L4=认知策略填充 |
| 结构模板 (articleStructureTemplate) | 增强 | 结构模板=通用结构，数字资产L2=个性化结构指导 |
| 用户观点 (userOpinion) | 独立 | 用户观点=本次创作约束，数字资产=历史风格参考 |

**核心原则**：数字资产是「风格参考」而非「硬性约束」，当与用户观点/创作类型冲突时，以用户显式输入为准。
