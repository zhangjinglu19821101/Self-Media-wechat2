## 二、已实施优化详细对比

### 优化1: 防抖自动保存Hook

#### 优化前 (`page-v2.tsx`)

```typescript
// ⚠️ 问题1: 无防抖，每次输入都保存
useEffect(() => {
  try {
    localStorage.setItem(STORAGE_KEY, JSON.stringify(data));
  } catch (error) {
    console.warn('保存草稿失败:', error);
  }
}, [coreAnchorData, materialData, creationControlData]);
```

**问题分析**:
- ❌ 用户每次输入都会触发保存（可能100ms保存一次）
- ❌ localStorage是同步操作，频繁调用可能造成UI卡顿
- ❌ 无变化时也会重复保存
- ❌ 缺少保存状态反馈
- ❌ 缺少强制保存机制

#### 优化后 (`src/hooks/use-debounced-storage.ts`)

```typescript
// ✅ 优化1: 防抖自动保存Hook
export function useDebouncedStorage<T extends object>(
  key: string,
  data: T,
  delay: number = 1000
): { isSaving: boolean; lastSaved: Date | null; forceSave: () => void } {
  
  // ✅ 优化2: 使用useMemo缓存序列化
  const serializedData = useMemo(() => {
    return JSON.stringify({
      ...data,
      savedAt: Date.now()
    });
  }, [data]);

  // ✅ 优化3: 无变化时不保存
  const prevDataRef = useRef<string>('');
  if (serializedData === prevDataRef.current) {
    return;
  }

  // ✅ 优化4: 防抖延迟
  timeoutRef.current = setTimeout(() => {
    localStorage.setItem(key, serializedData);
  }, delay);

  // ✅ 优化5: 提供保存状态反馈
  return {
    isSaving,        // 显示"保存中..."状态
    lastSaved,       // 显示"最后保存于..."
    forceSave        // 提供立即保存按钮
  };
}
```

**性能提升**:
- ⚡ 保存频率从"每次输入"降低到"输入停止后1秒"
- ⚡ 无变化时完全避免保存
- ⚡ 用户感知：不再感到输入时的卡顿
- ✅ 提供保存状态反馈（显示"保存中..."、"最后保存于..."）
- ✅ 提供强制保存按钮（重要场景用）

**代码行数**:
- 优化前: 约15行
- 优化后: 约120行（包含完整的错误处理、状态管理）

#### 优化前 (`core-anchor-input.tsx`)

```typescript
// ⚠️ 问题3: 常量硬编码在组件内部
```

**问题分析**:
- ❌ 验证逻辑重复调用（初始化 + useEffect）
- ❌ 验证函数、子组件、常量都在组件内部
- ❌ 组件难以单元测试（无法独立测试验证逻辑）
- ❌ 每次重新渲染都会创建新的子组件函数

#### 优化后 (`core-anchor-input-optimized.tsx`)

```typescript
// ✅ 优化1: 验证函数抽离到组件外（纯函数）
export function validateWordCount(
  text: string,
  min: number,
  max: number
): ValidationResult {
  const count = text.trim().length;
  const valid = count >= min && count <= max;
  
  // ✅ 优化2: 返回详细的验证信息
  return { 
    valid, 
    current: count, 
    message: valid ? undefined : '建议150-300字'
  };
}

// ✅ 优化3: 批量计算函数
export function computeCoreAnchorValidation(
  data: CoreAnchorData
): Record<keyof CoreAnchorData, ValidationResult> {
  return {
    openingCase: validateWordCount(data.openingCase, 150, 300),
    // ...
  };
}

// ✅ 优化4: 子组件也抽离到外部
export function WordCountIndicator({...}) {...}

// ✅ 优化5: 常量抽离到types.ts
import { WORD_COUNT_RANGES } from './types';

// ✅ 优化6: 使用useMemo缓存结果
export function CoreAnchorInput({ value, onChange }: CoreAnchorInputProps) {
  const validation = useMemo(() => {
    return computeCoreAnchorValidation(value);
  }, [value]); // 只在value变化时重新计算
}
```

**优化收益**:
- ⚡ 验证逻辑只在 value 真正变化时计算（useMemo）
- ✅ 纯函数可以独立单元测试
- ✅ 组件职责更单一（只负责UI渲染）
- ✅ 避免子组件函数重复创建
- ✅ 常量集中管理，易于修改

**可测试性提升**:
- 之前: 需要渲染整个组件才能测试验证逻辑
- 现在: 直接调用 `validateWordCount()` 纯函数进行测试

### 优化3: 大纲模板字符串注入安全

#### 优化前 (`page-v2.tsx`)

```typescript
// ⚠️ 问题1: 模板字符串直接拼接用户输入
```

**问题分析**:

- ⚠️ 如果用户输入包含特殊字符，可能造成XSS（虽然React已转义）
- ❌ 没有安全截断机制
- ❌ 没有输入验证
- ❌ 硬编码在页面组件中

#### 优化后 (`safety-utils.ts`)

```typescript
// ✅ 优化1: 安全截断函数
export function safeTruncate(
  text: string,
  maxLength: number,
  suffix: string = '...'
): string {
  if (!text) return '';
  if (text.length <= maxLength) return text;
  return text.substring(0, maxLength - suffix.length) + suffix;
}

// ✅ 优化2: 安全净化函数
export function sanitizeTemplateInput(text: string): string {
  if (!text) return '';
  
  return text
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/&/g, '&amp;')
    // ... 更多净化规则
}

// ✅ 优化3: 安全的大纲生成（使用数组而非字符串拼接）
export function generateSafeOutline(data: OutlineTemplateData): string {
  const truncatedOpening = safeTruncate(data.openingCase, 50);
  const truncatedViewpoint = safeTruncate(data.coreViewpoint, 50);

  // 使用数组构建，避免字符串拼接
  const outlineLines = [
    '【创作大纲】',
    '',
    `1. 真实故事/案例开头（约300字）`,
    `   - 核心素材：${truncatedOpening}...`,
    // ...
  ];

  return outlineLines.filter(line => line.trim() !== '').join('\n');
}

// ✅ 优化4: 安全的JSON工具
export function safeJsonParse<T>(text: string, defaultValue: T): T {...}
export function safeJsonStringify(data: object, space?: number): string {...}

// ✅ 优化5: 输入验证工具
export function validateInput(
  value: string,
  options: InputValidationOptions = {}
): InputValidationResult {...}

// ✅ 优化6: 存储大小检查
export function checkStorageSize(data: object): {...}
```

**安全提升**:
- ✅ XSS防护双重保障（React转义 + 安全净化）
- ✅ 安全截断防止输出过长
- ✅ 输入验证确保数据符合预期
- ✅ 存储大小检查防止超出限制
- ✅ 安全的JSON解析/序列化防止崩溃
