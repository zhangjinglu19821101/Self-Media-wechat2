# AI写作风格复刻系统功能设计 — 像人一样思考

> **文档版本**：v1.0  
> **创建日期**：2025-07-11  
> **核心命题**：当前数字资产对「像人写」的贡献约25-30%，如何提升到70-80%？

---

## 一、问题诊断：为什么当前方案「像人」程度有限

### 1.1 当前系统做了什么

当前系统从标杆文章中提取两类数据：

```
Step 1：范式识别（10套标准范式，5维匹配）
  → 输出：匹配的范式ID、结构映射、匹配度

Step 2：关系型素材提取（7维）
  → 输出：错误认知、生活类比、真实案例、权威数据、金句、固定句式组合、个人碎片
  → 每条素材含：内容、前文、后文、功能标签、情绪、范式步骤
```

### 1.2 诚实评估：各维度对「像人写」的实际贡献

| 维度 | 提取了什么 | 对「像人写」贡献 | 原因 |
|------|-----------|-----------------|------|
| 范式识别 | 叙事结构 | **中低** | 知道结构≠会按结构自然展开 |
| 错误认知 | 说了什么 | **低** | 知道观点≠知道为什么会想到这个观点 |
| 生活类比 | 比喻内容 | **中** | 有帮助，但AI构造类比≠人联想到类比 |
| 真实案例 | 案例内容 | **中低** | 案例可用，但AI无法复刻"回忆起这件事"的语气 |
| 权威数据 | 数据本身 | **低** | 纯信息，不影响写作风格 |
| 金句 | 好句子 | **中低** | 可复用，但套用金句≠自然说出金句 |
| 固定句式组合 | 标志性表达 | **中高** | 最接近「像人」的维度，直接复刻表达习惯 |
| 个人碎片 | 语气词/停顿 | **中高** | 最有「人味」的维度，但当前注入提示词时权重最低 |

**当前总贡献**：约 25-30%

### 1.3 根本原因：3个结构性缺失

```
像人写的文章 = 思维路径（40%） + 情感节奏（35%） + 语言习惯（25%）

当前系统覆盖情况：
  ✅ 语言习惯 → 固定句式+个人碎片覆盖了约60%
  ⚠️ 思维路径 → 范式识别覆盖了约20%（只知结构，不知路径）
  ❌ 情感节奏 → 几乎未覆盖（有emotionCurve但未注入创作）
```

#### 缺失1：思维路径（贡献最大，覆盖最少）

```
人写文章的内心过程：
  看到客户的原话 → 心里一阵不舒服 → 想起自己的经历 →
  觉得客户的想法有道理但又不对 → 突然想到一个生活类比 →
  类比让自己想通了 → 把"想通的过程"写出来

AI写文章的过程：
  收到指令+素材 → 按模板填空 → 输出

差距：AI没有"心里不舒服→想起经历→类比想通"这个思维过程
```

**举例**：

```
❌ 当前方案：
  素材库告诉AI："闹钟比喻：闹钟的价值不是响了才有用"
  AI直接用这个比喻，但不知道为什么要用这个比喻
  结果：比喻出现了，但没有"想到"的过程，读起来像硬塞的

✅ 应该做到的：
  AI知道：作者在反驳"回本逻辑"时，是因为自己2025年肺部手术的经历，
  才自然联想到"买了保险没用到不等于亏了"——而闹钟比喻是从这个经历中
  自然生发出来的，所以比喻出现在"自我经历"之后，语气是释然的
```

#### 缺失2：情感节奏（完全未注入创作）

```
人写文章的情绪曲线：
  焦虑 ──→ 共情 ──→ 转折 ──→ 释然 ──→ 坚定
  "每年交3000"  "换谁都会"  "但它犯了"  "闹钟比喻"  "恭喜你"
  
  特征：每段语气不同，像在跟朋友聊天，有情绪起伏

AI写文章的情绪曲线：
  平 ──→ 平 ──→ 平 ──→ 平 ──→ 平
  每段都是"专业科普"的语气，没有情绪起伏
  
  特征：全程一种语气，像在写说明书
```

当前系统已经提取了 `emotionCurve`（段落的情绪标注序列），但**从未注入到创作提示词中**。

#### 缺失3：示例引导 vs 规则约束

```
当前的注入方式（规则约束）：
  "禁用：综上所述、值得注意的是"
  "必须使用：口语化表达"

问题：规则只能约束"不做什么"，不能引导"怎么做"

更好的方式（示例引导）：
  "参考这个段落的写法：
   '但我得说——换谁都会这么想。你说交了9万只保10万，
   算上利息还亏了。这个算法，逻辑上没问题。'
   注意：先承认→再转折，语气像在和朋友聊天，不是在写论文"

核心：一个具体示例 > 十条抽象规则
```

## 三、Layer 1：思维路径复刻

### 3.1 核心概念

**思维路径**：作者从"看到什么"到"想到什么"再到"写出什么"的完整思维过程。

```
当前的提取（描述"写了什么"）：
  素材类型：生活类比
  内容："就像你买了一个闹钟——闹钟的价值不是响了才有用"

应该增加的提取（描述"怎么想到的"）：
  thinkingPath: {
    trigger: "客户说'交了9万只保10万，亏了'",
    feeling: "觉得客户不是对保险有意见，是对'没用到'有意见",
    association: "想到自己住院时听到隔壁床闹钟响",
    turningPoint: "意识到闹钟的本质是'安心'，不是'响了才有用'",
    output: "闹钟类比自然生发出来"
  }
```

### 3.2 数据结构设计

**方案**：在 `extraction_assets` 表的 `asset_data` JSONB 字段中，为每条关系型素材新增 `thinkingPath` 子结构。

```typescript
// 扩展 RelationalMaterial 接口
interface RelationalMaterialWithThinking extends RelationalMaterial {
  // 新增：思维路径
  thinkingPath?: {
    /** 触发点：看到/听到什么，才想到写这个素材 */
    trigger: string;
    /** 内心感受：想到这个素材时的情绪/直觉 */
    feeling: string;
    /** 联想来源：从什么联想到这个素材（经历/类比/数据） */
    association: string;
    /** 转折点：这个素材让作者想通了什么 */
    turningPoint: string;
    /** 输出方式：这个素材在文中是如何自然出现的 */
    outputStyle: '直接引用' | '自然生发' | '转折引出' | '反问引出';
  };
}
```

### 3.3 提取提示词扩展

在 `buildRelationalExtractionPrompt` 中，为每个维度的提取规则增加「思维路径」提取指令：

```
### 2. 生活类比
- 定义：所有用生活化事物解释保险概念的比喻，以及引出比喻的前一句话
- 提取规则：...（现有规则不变）
- 【新增】思维路径提取规则：
  对于每个提取到的类比，请推断作者是怎么想到这个类比的：
  - trigger：作者看到/听到了什么，才联想到这个比喻？
  - feeling：联想时的内心感受是什么？（如释然、恍然、无奈）
  - association：从什么事物联想到的？（真实经历/日常场景/其他文章）
  - turningPoint：这个比喻让作者想通了什么？
  - outputStyle：这个比喻在文中是怎样出现的？
    - 直接引用：开头就抛出比喻
    - 自然生发：从上文内容自然过渡到比喻
    - 转折引出：用"但"、"不过"等转折词引出
    - 反问引出：用反问句引出比喻
```

### 3.4 注入创作的方式

```typescript
// PromptAssemblerService 新增方法
formatThinkingPathForPrompt(materials: RelationalMaterialWithThinking[]): string {
  if (!materials.length) return '';
  
  const paths = materials
    .filter(m => m.thinkingPath)
    .map(m => {
      const tp = m.thinkingPath!;
      return `当你要写"${m.content.slice(0, 30)}..."时：
  - 你是因为"${tp.trigger}"才想到写这段
  - 你当时的感受是：${tp.feeling}
  - 你从"${tp.association}"联想到了这个内容
  - 这个内容让你想通了：${tp.turningPoint}
  - 在文中${tp.outputStyle}`;
    })
    .join('\n\n');

  return `## 思维路径指引

你不是在按模板填空，你是在像人一样思考。以下是每个关键段落的思维过程：

${paths}

写作时请遵循这些思维路径——先想到trigger，产生feeling，再联想association，最后自然写出内容。不要跳过思维过程直接输出结果。`;
}
```

## 四、Layer 2：情感节奏复刻

### 4.1 核心问题

当前系统已提取 `emotionCurve`（段落的情绪标注+强度序列），但**从未注入到创作提示词中**。

```typescript
// 已有数据结构（article-extraction-service.ts）
emotionCurve: Array<{
  paragraphIndex: number;
  paragraphPreview: string;
  emotion: string;       // "焦虑"/"共情"/"释然"/"坚定"
  intensity: number;     // 1-10
}>;
```

这些数据已经在数据库里了，只是没有用到创作中。

### 4.2 情感节奏的3个层次

```
层次1：段落级情绪（已有，未注入）
  → 告诉AI"每段应该有什么情绪"

层次2：段落间情绪过渡（需新增提取）
  → 告诉AI"从焦虑到共情，怎么自然过渡"

层次3：情绪对应的语气变化（需新增提取）
  → 告诉AI"焦虑时语气短促，释然时语气舒缓"
```

### 4.3 数据结构设计

```typescript
// 新增：情感节奏扩展数据
interface EmotionRhythmExtended {
  // 层次1：段落级情绪（已有，复用 emotionCurve）
  paragraphEmotions: Array<{
    paragraphIndex: number;
    paragraphPreview: string;
    emotion: string;           // 情绪标签
    intensity: number;         // 强度 1-10
  }>;

  // 层次2：情绪过渡模式（新增）
  emotionTransitions: Array<{
    fromEmotion: string;       // 前一段情绪
    toEmotion: string;         // 后一段情绪
    transitionPhrase: string;  // 过渡句原文（如"但我想请你冷静下来想一想"）
    transitionType: '转折' | '递进' | '并列' | '回扣';
  }>;

  // 层次3：情绪-语气映射（新增）
  emotionToneMap: Array<{
    emotion: string;           // 情绪标签
    toneDescription: string;   // 语气描述
    sentenceLength: '短促' | '中等' | '舒展';
    exampleSentence: string;   // 原文中的示例句
  }>;
}
```

### 4.5 注入创作的方式

```typescript
// PromptAssemblerService 新增方法
formatEmotionRhythmForPrompt(emotionRhythm: EmotionRhythmExtended): string {
  if (!emotionRhythm.paragraphEmotions.length) return '';

  // 层次1：情绪曲线
  const curve = emotionRhythm.paragraphEmotions
    .map(e => `${e.emotion}(${e.intensity}/10)`)
    .join(' → ');

  // 层次2：过渡模式
  const transitions = emotionRhythm.emotionTransitions
    .map(t => `${t.fromEmotion} → ${t.toEmotion}：用"${t.transitionPhrase}"过渡（${t.transitionType}）`)
    .join('\n');

  // 层次3：语气指引
  const toneMap = emotionRhythm.emotionToneMap
    .map(t => `${t.emotion}：${t.toneDescription}（${t.sentenceLength}句式）\n  示例："${t.exampleSentence}"`)
    .join('\n\n');

  return `## 情感节奏指引

你不是在写说明书，你是在跟读者对话。文章的情绪应该有起伏。

### 情绪曲线
${curve}

### 情绪过渡方式
${transitions}

### 每种情绪的语气要求
${toneMap}

请严格按照这个情绪曲线写作——该焦虑时句子短促，该释然时语气舒展，该坚定时掷地有声。不要全程一种语气。`;
}
```

### 4.6 示例输出

```
## 情感节奏指引

你不是在写说明书，你是在跟读者对话。文章的情绪应该有起伏。

### 情绪曲线
焦虑(7/10) → 共情(5/10) → 突破(8/10) → 释然(6/10) → 坚定(7/10)

### 情绪过渡方式
焦虑 → 共情：用"说实话，看到这样的新闻，我心里也不好受"过渡（转折）
共情 → 突破：用"但我想请你冷静下来想一想"过渡（转折）
突破 → 释然：用"就像你买了一个闹钟"过渡（递进）
释然 → 坚定：用"如果这事发生在你身上"过渡（反问回扣）

### 每种情绪的语气要求
焦虑：句子短促，用问句制造紧迫感，断句多（短促句式）
  示例："我每年交3000块，交30年就是9万。你才给我保10万，算上利息我还亏了——保险就是骗人的。"

共情：语气缓和，用'说实话'、'换谁都会'拉近距离（中等句式）
  示例："这种想法，逻辑上没问题。"

释然：语气舒展，用类比讲道理，像在跟朋友聊天（舒展句式）
  示例："就像你买了一个闹钟——闹钟的价值不是响了才有用。设了闹钟之后你能睡得踏实，这本身就是它最大的价值。"

坚定：掷地有声，用反问收尾，不留退路（短促句式）
  示例："如果这事发生在你身上，你真会觉得一大笔大病理赔是'赚到'了吗？"

请严格按照这个情绪曲线写作——该焦虑时句子短促，该释然时语气舒展，该坚定时掷地有声。不要全程一种语气。
```

## 五、Layer 3：语言习惯复刻 — 从规则约束升级为示例引导

### 5.1 核心问题

当前的 `styleAssets`（风格规则）以抽象规则为主：

```
当前注入方式：
  规则类型：禁用词
  规则内容：禁用"综上所述"、"值得注意的是"、"不可否认"

问题：
  1. AI不用这些词，但换成"总的来说"，依然有AI味
  2. 规则只能约束"不做什么"，不能引导"怎么做"
  3. 同一条规则在不同上下文中效果不同
```

### 5.2 改进方案：示例引导

**核心原则**：一个具体示例 > 十条抽象规则

```typescript
// 当前方式（规则约束）
interface StyleRule {
  ruleType: 'banned_words' | 'required_style' | ...;
  ruleContent: string;   // "禁用：综上所述"
}

// 改进方式（示例引导）
interface StyleGuide {
  // 保留规则（兜底约束）
  ruleType: string;
  ruleContent: string;
  
  // 新增：示例引导（主要注入方式）
  examples: Array<{
    context: string;        // 使用场景："承认对方观点时"
    badExample: string;     // AI味写法："综上所述，保险的价值不容忽视"
    goodExample: string;    // 人味写法："但我得说——换谁都会这么想"
    why: string;            // 原因："先承认→再转折，语气像在和朋友聊天"
  }>;
}
```

### 5.3 示例数据

```typescript
const STYLE_GUIDES = [
  {
    context: '承认对方观点时',
    badExample: '诚然，这种观点有一定的合理性，但从专业角度分析……',
    goodExample: '这种想法，逻辑上没问题。',
    why: '直接肯定，不绕弯，不堆砌连接词'

  {

  },

  }

```

### 5.4 注入方式

```typescript
formatStyleGuidesForPrompt(guides: StyleGuide[]): string {
  if (!guides.length) return '';

  const examples = guides
    .map(g => `场景：${g.context}
  ❌ AI味：${g.badExample}
  ✅ 人味：${g.goodExample}
  原因：${g.why}`)
    .join('\n\n');

  return `## 写法示范（比规则更重要）

以下是5种常见场景的写法对比。请参考✅人味的写法，避免❌AI味的写法：

${examples}

关键原则：你是在跟朋友聊天，不是在写论文。直接说，不要绕弯。`;
}
```

### 5.5 提取方式

从 `fixed_phrase`（固定句式组合）和 `personal_fragment`（个人碎片）中自动生成示例引导：

```typescript
function generateStyleGuidesFromExtractions(
  materials: RelationalMaterial[]
): StyleGuide[] {
  const guides: StyleGuide[] = [];
  
  // 从固定句式组合生成
  const fixedPhrases = materials.filter(m => m.materialType === 'fixed_phrase');
  for (const fp of fixedPhrases) {
    guides.push({
      context: fp.sceneTags.join('、') || '通用',
      badExample: generateAIStyleEquivalent(fp.content), // AI写法的等价表达
      goodExample: fp.content,
      why: `这是作者的标志性表达，原文保留`
    });
  }
  
  // 从个人碎片生成
  const personalFragments = materials.filter(m => m.materialType === 'personal_fragment');
  for (const pf of personalFragments) {
    guides.push({
      context: '自然穿插个人化表达',
      badExample: '（无个人化表达）',
      goodExample: pf.content,
      why: '这种只有作者本人才会说的表达，是最有"人味"的部分'
    });
  }
  
  return guides;
}
```

### 6.2 注入优先级

```

冲突处理：
  - 用户选了「理性客观」基调 → 情感节奏降级为"微起伏"，不强制大起大落
  - 用户提供了核心观点 → 思维路径中与用户观点矛盾的部分自动跳过
  - 用户选择了特定素材 → 素材内容优先于数字资产中的同类素材
```

### 6.3 长度控制

| 注入内容 | 最大长度 | 占总提示词比例 |
|---------|---------|--------------|
| 思维路径指引 | 400字 | ~2% |
| 情感节奏指引 | 500字 | ~2.5% |
| 写法示范 | 300字 | ~1.5% |
| **合计** | **1200字** | **~6%** |

控制策略：
- 思维路径：只取匹配度最高的3条素材的路径
- 情感节奏：只取5种核心情绪的映射
- 写法示范：只取5种最常见场景的对比

## 十、总结

### 一句话

> 当前方案是在告诉AI"不要这样写"，真正需要的是告诉AI"人是怎么想到要这样写的"。

### 三个核心改进

1. **思维路径**：让AI知道"为什么要写这段"，而不只是"写什么"
2. **情感节奏**：让AI知道"写这段时应该有什么情绪"，而不只是"写什么内容"
3. **示例引导**：用"人味写法 vs AI味写法"的对比示范，替代抽象规则

### 实施优先级

```
Phase 1（情感节奏）← 最小改动 + 最大收益，优先实施
Phase 2（思维路径）← 核心差异化能力
Phase 3（示例引导）← 锦上添花
Phase 4（提示词改造）← 整合收尾
```
