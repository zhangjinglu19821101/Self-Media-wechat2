# AI写作风格复刻系统功能设计需求文档

# 一、项目概述

1. 项目名称：AI写作风格复刻系统（适配insurance-d）

2. 项目版本：V1.0

3. 核心目标：不训练大模型，仅通过“提示词动态拼接+本地知识沉淀”，实现AI写作风格100%复刻用户，达成“越用越像、越用越准、越用越好”的效果；同时满足用户核心创作诉求，让AI严格遵循用户设定的核心观点、固定结构和关键素材，不偏离用户写作逻辑与风格。

4. 核心约束：AI不可篡改用户核心思想、严格遵循用户选定的固定结构、仅使用用户提供的关键素材，全程贴合用户写作语气、用词习惯和表达逻辑。

5. 适用场景：用户（保险科普类公众号创作者）通过系统输入核心内容，让AI快速生成符合自身风格的文章，降低创作成本，同时保持文章风格统一性。

# 二、用户核心期望与系统功能对应要求

## 2.1 用户核心期望

- 期望1：可表达每篇文章核心观点，包括开篇核心案例段、全文核心观点段、结尾核心结论段，且可补充关联素材供AI参考。

- 期望2：可选择每篇文章的固定结构（用户专属7段结构），AI严格按该结构顺序创作，不跳步、不删减。

- 期望3：创作前可提供本篇文章所需关键素材，AI优先使用该素材，不编造无依据内容。

- 核心目标：AI写作风格逐步贴合用户，越用越像，无需人工大量修改即可使用。

## 2.2 系统核心功能定位

围绕用户期望，系统需实现“核心内容输入+固定结构选择+素材提供+知识沉淀+提示词动态增强”五大核心功能，无需训练大模型，仅通过本地数据沉淀和提示词优化，实现AI风格复刻的持续优化。

# 三、系统功能设计（核心回答用户3个问题）

## 3.1 界面功能设计（贴合用户操作习惯，极简高效）

界面核心目标：让用户快速输入核心内容、选择结构、提供素材，操作流程简单，无多余步骤，同时支持知识沉淀的查看与管理。

### 3.1.1 核心锚点输入模块（必填，锁死文章灵魂）

|字段名称|控件类型|字数限制|功能说明|
|---|---|---|---|
|开篇核心案例段|多行文本框|150-300字|用户亲笔输入，需包含真实人物、场景、情绪，AI必须原样使用，不可修改、替换|
|全文核心观点段|多行文本框|100-200字|用户输入核心立场、最终结论及不可改写的红线，AI严格遵循，不可反向解读|
|结尾核心结论段|多行文本框|100-200字|用户输入最终避坑建议、态度及收尾观点，AI仅可润色细节，不可修改结论|
### 3.1.2 关联素材与关键素材提供模块（可选+可选填）

|字段名称|控件类型|功能说明|
|---|---|---|
|关联素材补充区|多行文本框|用户粘贴关联素材（知识点、补充案例等），AI优先参考使用|
|本篇关键素材区|多行文本框|用户输入本篇文章专属关键素材（权威数据、条款、案例细节等），AI仅可使用该素材，不编造内容|
### 3.1.3 固定结构选择模块（核心骨架）

|字段名称|控件类型|默认值|功能说明|
|---|---|---|---|
|选择文章结构|下拉单选框|用户专属7段固定结构|仅展示用户保存的固定结构，不可编辑；选中后实时展示结构明细，供用户确认|
|结构明细展示区|只读文本区|-|固定展示用户专属7段结构：1.真实故事/案例开头（身边人、真实事）；2.抛出用户最关心的疑问；3.情绪/立场表达（共情、不平、惋惜、替用户说话）；4.理性拆解真相（纠正误区）；5.权威数据/规则支撑；6.给普通人可落地的避坑建议；7.结尾互动+合规声明|
### 3.1.4 创作控制与大纲确认模块（防止AI跑偏）

|字段名称|控件类型|功能说明|
|---|---|---|
|目标字数|输入框|默认2000-2200字，浮动±200字，严格控制文章篇幅，贴合用户写作习惯|
|生成大纲按钮|功能按钮|点击后，AI基于用户输入的核心锚点、素材和固定结构，生成创作大纲，供用户确认|
|大纲确认区|只读文本框|展示大纲明细（结构+核心观点+素材使用规划），用户可核对，确认无误后才可触发全文生成|
|生成全文按钮|功能按钮|仅大纲确认后可点击，AI按大纲、提示词和用户需求生成全文|
### 3.1.5 风格资产管理模块（知识沉淀入口）

用于用户查看、编辑、管理本地沉淀的数字资产，助力AI越用越像用户，核心功能如下：

- 风格习惯管理：查看、编辑用户高频词、禁用词、语气规则、排版习惯

- 核心观点库：查看历史所有核心锚点（开篇案例、核心观点、结尾结论），支持搜索、复用

- 结构模板库：查看、管理用户固定结构，支持新增自定义结构（需贴合用户写作逻辑）

- 素材库：查看、管理用户常用关键素材、权威数据、案例，支持分类、标签化管理

- 反馈修正库：查看用户对AI文章的修改记录、优化要求，支持手动添加AI写作禁令

## 3.2 insurance-d系统提示词设置（核心，确保AI复刻风格）

提示词核心逻辑：固定基础规则+动态沉淀资产+本次创作需求，三者拼接，确保AI严格遵循用户要求，且每次使用都能贴合最新沉淀的用户风格，无需训练模型。

### 3.2.1 提示词固定基础部分（不可修改）

```plain text
你是【智者足迹-探寻】专属保险文案AI，严格遵守以下规则，100%复刻用户写作风格，不偏离用户核心思想，不编造任何无依据内容：
【一、核心铁律（绝对不可违反）】
1.  必须完整使用用户提供的：开篇核心案例段、全文核心观点段、结尾核心结论段，不得修改、替换、删减、反向解读，仅可对结尾结论做细节润色（不改变原意）。
2.  必须严格按照用户选定的固定文章结构，按顺序写作，不得跳步、调换结构顺序、删减结构模块，每个模块需贴合用户设定的功能要求。
3.  必须优先使用用户提供的关联素材和本篇关键素材，不编造数据、案例、保险条款，无依据的内容绝对禁止出现。
4.  禁止使用任何绝对化、营销类词汇（包括但不限于：最、第一、100%、保本、稳赚、绝对），禁止堆砌专业术语，所有专业内容需用大白话解读。

【二、风格复刻基础要求】
1.  口吻：第一人称「我」，称呼用户为「你/咱们」，语气共情、亲切，像朋友掏心窝聊天，不端着、不偏激。
2.  排版：短句为主，每段1-3行，手机阅读无压力，避免大段文字堆砌。
3.  人设：站在消费者立场，不推荐任何具体保险产品，专业理性有温度，不制造焦虑、不夸大风险。
4.  篇幅：严格控制在用户设定的目标字数范围内，结构占比贴合用户习惯（案例30%+拆解40%+建议20%+互动合规10%）。

【三、创作流程要求】
1.  先根据用户输入的核心锚点、素材、固定结构，生成创作大纲，等待用户确认后，再生成完整文章。
2.  大纲需清晰呈现：结构模块、每个模块的核心内容、素材使用规划，确保贴合用户核心思想。
3.  全文禁止使用通用AI套话，禁止偏离用户写作逻辑，禁止出现与用户风格不符的表述。
```

### 3.2.2 提示词动态拼接部分（系统自动生成，随沉淀资产更新）

系统每次生成全文前，自动从本地沉淀的数字资产中提取用户专属规则，拼接至基础提示词后，形成最终专属提示词，示例如下：

```plain text
【用户专属动态规则（自动拼接，随使用更新）】
1.  用户高频用词：踩坑、门道、算笔明白账、兜底、先讲真相、先说结论（需高频使用，贴合用户表达习惯）。
2.  用户禁用补充：禁止使用「您、客户」等称呼，禁止过度煽情，禁止简化核心拆解逻辑。
3.  用户核心立场补充：保险是保障不是理财，普通人买保险优先看杠杆率，不推荐返还型保险（提取用户历史观点沉淀）。
4.  用户固定结构补充：每个结构模块需贴合用户习惯，案例部分需突出真实感，情绪表达需共情用户，数据部分需通俗解读。
5.  用户素材使用习惯：权威数据优先引用国家金融监管总局、国家癌症中心发布的内容，案例优先使用身边匿名真实案例。
```

### 3.2.3 提示词拼接规则

最终提示词 = 固定基础提示词 + 用户专属动态规则 + 本次创作需求（核心锚点、素材、结构、目标字数），系统自动拼接，无需用户手动操作，每次使用都会更新动态规则部分，确保AI越用越像。

## 3.3 需重点沉淀的数字资产（AI越用越像的关键）

所有资产本地存储、自动更新，不训练模型，仅用于动态拼接提示词，核心分为5大类，确保资产可复用、可迭代，逐步贴合用户写作风格。

### 3.3.1 风格资产（复刻用户语气、用词、排版的核心）

- 高频词：用户写作中反复使用的词汇（如：踩坑、门道、算笔明白账、兜底等），系统自动统计、更新。

- 禁用词：用户明确禁止使用的词汇，以及系统统计的用户从未使用过的绝对化、营销类词汇。

- 语气与句式习惯：用户常用句式（如：反问句、短句、“咱们先算笔账”等）、情绪表达偏好（共情为主、理性拆解为辅）。

- 排版规则：用户固定的段落长度（每段1-3行）、标题层级、字数占比等。

### 3.3.2 核心锚点资产（用户思想与立场的沉淀）

- 历史核心锚点：用户每篇文章输入的开篇核心案例段、全文核心观点段、结尾核心结论段，按时间排序，支持搜索、复用。

- 观点红线：用户反复强调的核心立场、不可改写的观点（如：保险是保障不是理财、不推产品等）。

- 模板素材：用户常用的结尾互动模板、合规声明模板，自动沉淀、复用。

### 3.3.3 结构资产（文章骨架的沉淀）

- 固定结构模板：用户专属7段结构的详细说明、字数占比，确保AI每次创作都贴合该结构。

- 结构使用数据：用户使用各结构的频次，优化结构优先级，贴合用户使用习惯。

- 自定义结构：用户后续新增的固定结构，统一存储、管理，支持一键选择。

### 3.3.4 素材资产（创作依据的沉淀）

- 权威素材：用户常用的权威数据、保险条款、官方政策等，按类型分类，支持快速调用。

- 案例素材：用户输入的真实案例、关联案例，自动沉淀，供AI后续创作参考、复用。

- 素材使用规则：用户偏好的素材使用方式、数据解读逻辑，自动统计、更新，确保AI使用素材的方式贴合用户习惯。

### 3.3.5 反馈迭代资产（AI越用越好的关键）

- 修改记录：用户对AI生成文章的修改内容、不满意点，系统自动记录，提取优化规则。

- AI写作禁令：用户手动添加或系统自动提取的禁止性规则（如：禁止过度煽情、禁止简化拆解逻辑等）。

- 迭代日志：资产更新记录、规则优化记录，可追溯、可回滚，确保资产迭代可控。

# 四、资产沉淀机制（AI越用越像的核心逻辑）

核心原理：不训练大模型，仅通过“提示词动态增强”实现风格复刻，用户使用次数越多，沉淀资产越全，提示词越贴合用户，AI写作风格越像。

## 4.1 自动沉淀逻辑

- 每篇文章的核心锚点，自动存入核心锚点资产库，系统统计高频观点、常用表述。

- 用户每次修改AI文章，系统自动提取修改要点，更新反馈迭代资产库，转化为AI写作禁令。

- 每篇定稿文章，系统自动拆解其用词、句式、情绪，更新风格资产库，补充高频词、优化语气规则。

- 用户使用素材、结构的频次，自动统计，优化素材、结构的优先级，确保AI优先使用用户常用内容。

## 4.2 迭代机制

- 自动迭代：系统定期聚合沉淀的资产，优化提示词动态拼接规则，无需人工干预。

- 手动迭代：用户可通过风格资产管理模块，手动编辑、新增、删除资产规则，灵活调整AI写作风格。

# 五、配套校验规则（确保AI不跑偏）

结合Agent B校验能力，新增以下校验规则，优先级高于其他校验，确保AI严格贴合用户需求：

- 核心锚点校验：AI生成的文章必须100%保留用户输入的3段核心锚点，任何修改、替换都直接打回重写。

- 结构校验：必须严格按用户选定的固定结构顺序创作，缺模块、乱顺序、删减模块，直接打回重写。

- 风格校验：必须使用用户高频词、遵循用户语气和排版规则，禁用词出现即打回，自动修正。

- 素材校验：仅可使用用户提供的素材，禁止使用外部无依据素材，违规即打回。

# 六、核心创作流程（闭环，确保高效且不跑偏）

1. 用户操作：填写3段核心锚点 + 补充关联素材、关键素材 → 选择固定结构 → 设定目标字数。

2. AI操作：生成创作大纲（贴合核心锚点、素材、结构） → 等待用户确认。

3. 用户确认：核对大纲，确认无误后触发全文生成；若有误，可修改大纲后重新确认。

4. AI生成：系统自动拼接提示词（基础+动态资产+本次需求），AI按提示词生成全文。

5. 校验环节：Agent B按校验规则，先校验核心锚点、结构、风格、素材，通过后输出最终稿件。

6. 资产沉淀：系统自动将本次核心锚点、素材、修改记录（若有）沉淀至对应资产库，更新动态提示词规则。
